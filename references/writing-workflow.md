# 教育技术学 C 刊正文写作工作流

用于生成、改写或诊断论文正文。目标是让论文承担社会科学的解释、论证和对话功能，而不是把原始分析过程逐段复述。

## 一、写作前先确定四件事

1. **教育问题**：谁在何种学习、教学、评价或组织活动中遇到什么需要解释的现象？
2. **中心判断**：全文准备提出什么关系、过程、差异、模型或设计认识？
3. **论证材料**：理论、文献、数据、案例和图表分别承担什么作用？
4. **文类与读者**：这是理论、综述、实验、设计型、质性、混合方法还是学习分析论文？

先读 `academic-writing-and-revision.md` 搭主线，再按章节读取 `assets/section-prompts/`。不要在每次改写前强制生成研究风险、验证状态或方法审计。

实证、课堂观察、学习分析或机器学习研究再读取 `education-technology-empirical-writing-closure.md`：先让理论、构念、指标、表图、讨论与启示使用同一条主线，再组织段落。它不要求套用固定发现数量、结果小节数量或参考论文的表达。

## 二、论文正文与方法审计分开

论文正文写已经发生的研究、作者要提出的教育判断和能够回到材料的论证。方法审计仅在用户要求设计检查、复现或投稿预审时另列；它不自动成为摘要、讨论、结论或局限的材料。

清洁正文不得出现模式名、风险标签、待办、模板字段或“投稿时补充”等占位语。用户明确要求终稿时，事实缺失只写在稿外清单，不用原则性提示填补正文。

## 三、各章节承担的判断

| 章节 | 核心任务 | 常见失配 |
| --- | --- | --- |
| 题目与摘要 | 让读者迅速理解问题、研究行动、发现和推进 | 用算法名、指标和审计事项替代中心发现 |
| 引言 | 将现实场景、既有研究与可回答问题连成一条线 | 政策/技术热点代替问题意识 |
| 综述与理论 | 说明已有研究如何推进、为何不足，以及本文以何种理论进入 | 作者或概念罗列，理论只作装饰 |
| 方法 | 说明材料、研究决策和分析如何回答问题 | 软件、公式和算法教科书复述 |
| 结果 | 以表图报告经验发现并作最小判断 | 按代码顺序复述全部数值，抢先写理论贡献 |
| 讨论 | 将发现推回理论、文献和教育实践 | 只重复结果，或用长段方法自我否定替代解释 |
| 结论 | 回答题目，提炼研究推进与相称启示 | 再次罗列结果或以“仍需检验”结束全文 |

## 四、摘要：压缩研究判断

先写“问题—研究行动—发现簇—中心判断”。样本、方法和一个性能指标按需要保留，但不要罗列算法、SHAP/ALE、Bootstrap、消融或验证流程。学习分析和机器学习研究应先把变量翻译为教育构念与过程维度，再写发现和意义。具体规则见 `abstract-state-and-evidence-control.md`。

## 五、引言、综述与理论：把材料推向问题

- 引言从主体、活动、技术功能或规则、困难及其后果进入；政策背景只能说明情境。
- 综述按争议、解释、方法转向或情境差异组织，段末推出本文真正能回答的缺口。
- 理论必须进入研究对象、变量、编码、设计或解释；用它说明为什么这些材料能够共同回答问题。

不要默认以“近年来”“随着……发展”“本文首先……”起笔，也不要为形式整齐而强制编号研究问题。

## 六、方法与结果：让技术为论证服务

方法先说明研究材料和分析单位，再说明选择这种方法的理由与产生的证据。结果按研究问题或发现簇组织；表图已有的全部数值不在正文重复。

对机器学习研究：模型性能只是进入解释的基础，重要性、曲线和稳定性服务于对教育过程的理解。先将分析结果归并为构念或过程维度，再依照“基础比较—关键发现—关系形态/情境差异—必要的使用条件”的实际证据推进；没有某层证据时不强行补齐。正文可以使用“关键过程信息”“非线性关系”“过程配置”等学术表达；技术定义和复现细节留在方法、图注、表注或附录。只有用户要求方法审计时，才读取 `interpretable-machine-learning.md` 的审计部分。

## 七、讨论与结论：完成社会科学解释

讨论通常按“核心发现—理论/文献对话—教育过程解释—实践含义”推进。可以指出不同过程如何相互关联、共同呈现或构成某种教学/学习结构；不必反复把每一句降回“模型预测”。但并列变量并不自动构成时间链或因果机制：过程解释必须回到理论、材料和已报告的证据。

结论先直接回答题目，再选择两至三项发现支撑，说明相对于既有研究的最小推进和对实践的启示。局限只在需要时放在结尾，以一至两项最相关条件说明适用范围；不要将切分、调参、外部样本、伦理、未观测变量和未来研究逐项堆成数据分析报告。

## 八、反模板检查

- 删除空泛口号、对称清单、无对象的意义句和按模型步骤排列的叙述。
- 保留真实数据、专业术语、作者主语、必要的逻辑转折和已有理论对话。
- “本文”“本研究”“第一/第二”并非禁词；只有无真实功能、过密或机械对称时才重写。
- 不模仿特定作者或期刊的原句。使用功能性表达库时，先改写为本研究对象、材料和判断所必需的语言。
