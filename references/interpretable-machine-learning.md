# 教育技术学可解释机器学习方法

在使用惩罚回归、随机森林、梯度提升、XGBoost、SHAP、ALE、置换重要性或其他预测解释方法时读取。目标是建立可复现的“教育问题—数据生成—样本外预测—模型解释—教育含义”证据链，不把黑箱解释包装成因果识别。涉及理论特征的多级归类、维度汇总、PCA综合、分组排名、Bootstrap解释稳定性或删除节点实验时，同时读取 `hierarchical-ml-interpretation.md`。

## 1. 任务界定

- 明确预测对象、分析单位、目标变量、预测时点、特征可用时点和使用场景。区分连续结果、分类、序数结果、计数和生存/事件时间任务。
- 先说明为什么线性或理论模型不足，以及机器学习要处理的非线性、交互、高维或预测问题；不能以“算法先进”代替方法必要性。
- 把理论用于特征选择、特征分组、交互假设和结果解释。技术日志必须映射到学习者、教师、教学活动与情境。
- 预测研究直接回答“哪些信息有助于预测”，不自动识别“什么因素塑造、促进或导致结果”。但在理论、特征分组和观察框架明确时，可把稳定的发现归并为教育学的**解释性过程结构**，说明研究对象呈现出的关联图景；这一层是研究解释，不是净因果效应或已证实机制。需要因果问题时改用实验、准实验、纵向过程或明确的因果机器学习设计。

## 2. 数据审计与防泄漏处理

核心原则：先切分，后预处理；验证集和测试集不得参与任何参数学习或研究决策。

1. 建立数据字典：来源、采集时间、单位、量尺、缺失码、权限、去标识化、目标变量和潜在敏感属性。
2. 审核目标变量如何形成。若目标分数与预测特征来自同一量表、同一编码规则或包含重叠题项，先排除构念循环、同源评分和准则污染；高预测性能可能只是重建计分规则。
3. 在任何插补、标准化、编码、降维、特征选择或重采样前完成数据切分。所有需要 `fit` 的处理只在训练折中学习，并通过 pipeline 应用于验证/测试集。
4. 根据教育数据结构切分：同一学习者、班级、教师、学校或课程的记录不得无意跨训练与测试；纵向/预警任务按时间切分，测试集只能包含预测时点之后或真正未见的单位。数据缺少群组标识时标记限制，不能假定观测相互独立。
5. 排除目标泄漏、未来信息、目标的变体、干预后变量和预测时点不可获得的字段。记录删除理由，不只按相关系数筛选。
6. 缺失处理报告机制、比例与敏感性；插补器仅在训练折拟合。分类编码、量表合成、PCA 和其他特征工程同样放入折内。
7. 检查比例、时长和类别份额等组成数据。总和固定的整组比例具有结构依赖；不要把各分量当作彼此独立的可干预变量，可按问题采用参考类别、对数比、分组解释或敏感性分析。
8. 类别不平衡优先比较类别权重、阈值策略或训练折内重采样；保持验证/测试集的自然分布，并报告各类别样本量。
9. 保存原始/处理数据边界、代码本、切分索引、随机种子、软件版本和处理日志。涉及未成年人、平台日志或敏感属性时报告同意、最小化使用、公平性和再识别风险。

官方实践可核对 [scikit-learn 数据泄漏与 Pipeline 指南](https://scikit-learn.org/stable/common_pitfalls.html)。

## 3. 模型比较与验证

### 发布前的三段式验证设计

先把每一份数据的角色写入分析计划：训练集用于拟合与折内预处理；验证集或训练集内嵌套交叉验证用于调参、选模与解释模型选择；最终测试集只在模型、特征、阈值和报告口径锁定后使用一次。群组、课程、学校与时间边界优先于随机便利切分。

- 有独立最终测试且未参与任何研究决策：可申请 `投稿就绪`，但仍需检查准则污染、复现与结论边界。
- 只有训练集内验证或重复交叉验证：最多为 `探索性可写`，应报告验证方案而非把结果称为最终外部确认。
- 同一留出测试集比较多个候选模型后选择最高者，再报告该模型测试表现：`阻断`。需设置新测试集、使用嵌套验证，或明确降级为探索性模型筛选。

- 设置至少一个朴素基线和一个可解释基线，如均值/多数类、线性/逻辑回归或惩罚回归；再按任务选择随机森林、GBRT、XGBoost 等候选模型。不要默认算法越复杂越好。
- 固定独立测试集用于最终评价。调参只在训练数据内进行；样本较小或模型选择复杂时使用嵌套交叉验证。嵌套、群组和时间结构必须反映真实泛化目标。
- 比较大量算法时使用“训练—验证—最终测试”或嵌套交叉验证。若根据同一测试集上的20个模型排名选择解释模型，该数据已是模型选择集，不能再称为未触碰的最终测试集。
- 记录超参数搜索空间、选择指标、折数、重复次数、早停数据、种子和最终参数。测试集不得参与选模、阈值选择或早停。
- 回归至少报告基线、样本外 `R²`、MAE、RMSE及其不确定性；分类按任务报告 ROC-AUC、PR-AUC、灵敏度、特异度、F1、平衡准确率、Brier 分数和校准，不能只报准确率。
- 使用同一数据切分比较模型，并报告折间分布、置信区间或重复抽样结果。微小指标优势若落在不确定范围内，不宣称模型实质更优。
- 为性能接近的模型预先设定并列规则，结合误差区间、稳定性、校准、复杂度与解释需要选模，不按小数点后的偶然领先宣布唯一最优。
- 检查过拟合、校准、残差/误差分布、最差群体表现和外部/跨校/跨期泛化。若只完成内部验证，明确限制。

### 投稿发布门与局部承诺控制

以下问题属于分析修复项，不是可以只靠“局限说明”消解的写作问题：

- 目标分数直接由部分预测特征合成，或目标与特征共享评分规则而尚未做排除/敏感性分析；
- 根据同一测试集上多个模型的结果选出“最优模型”，又把该测试集性能称为最终样本外确认；
- 同一教师、班级、学校或课程的记录跨训练与测试，导致模型识别单位身份或共同环境；
- 插补、PCA、特征选择、编码或标准化在全数据上拟合；
- 主要模型、数据版本、切分索引或结果表无法复核。

未修复时，限制的是对应的强承诺，而不是教育学摘要本身：测试集选模只限制“独立最终最优表现”；曲线证据不足只限制“严格阈值”；没有因果识别只限制“行为导致结果”或保证性处方。它们不阻止围绕现有材料写出自然的教育问题、构念归并、解释性过程结构和有限的评价/改进意义。

用户要求摘要改写时，先完成摘要论证卡，再直接输出自然工作稿；仅在用户要求投稿判断，或风险确实改变稿中某一强承诺时，才在稿外另列“问题—修复—连锁影响”。草案中仍不得把同一留出集上的排序写为最终样本外表现。`投稿就绪` 是提交判断的门槛，不是自然教育学表述的门槛。任何状态都不把模型归因直接写成教育效果。

交叉验证与嵌套调参可核对 [scikit-learn 模型评估](https://scikit-learn.org/stable/modules/cross_validation.html)和[嵌套交叉验证示例](https://scikit-learn.org/stable/auto_examples/model_selection/plot_nested_cross_validation_iris.html)。

## 4. 可解释性分析

### SHAP

- 明确解释的模型、数据子集、模型输出尺度、背景数据和特征依赖处理方式。树模型可使用 TreeSHAP，但仍需报告设置。
- 全局平均绝对 SHAP 表示特征对该模型预测的平均贡献幅度，不是回归效应量、变量解释率或因果贡献率；绝对值排序不提供方向。
- 同时使用 beeswarm/分布图或依赖图呈现方向和异质性。局部 waterfall 仅解释具体预测，不把个案解释推广到总体。
- 高相关特征会分摊或改变归因；报告相关结构，必要时按理论分组解释、比较替代特征集或做稳定性分析。
- 跨学段或群体分别训练的模型，其平均 SHAP 不宜直接解释为“长期效应”或发展变化；先检验可比的特征、量尺、样本与模型性能，并使用“学段间预测重要性差异”。

树模型设置及相关特征假设可核对 [SHAP TreeExplainer 官方文档](https://shap.readthedocs.io/en/latest/generated/shap.TreeExplainer.html)。

### ALE 与其他效应图

- 特征相关明显时，ALE 通常比 PDP 少依赖训练分布之外的组合；它仍描述模型预测函数，不识别现实中的因果效应。
- 图中标出样本密度、分位点或 rug，并尽量用 bootstrap/重复拟合呈现不确定性。稀疏区间、边界和样本极少的分箱不得命名为稳定阈值。
- 若 ALE 曲线用于讨论拐点、平台或阈值，逐段报告有效样本量/密度、分箱规则和区间不确定性；尾部稀疏、置信区间宽或不同切分方向不一致时删除阈值叙述。
- PCA 合成变量的 ALE 只有在成分含义、载荷、缩放和可解释性充分报告时才可转化为教育含义。
- ICE 适合展示个体异质性；二维 ALE 或 SHAP interaction 只检验理论驱动的少量交互，避免遍历后挑选漂亮曲线。

ALE 的方法依据见 [Apley 与 Zhu（2020）](https://doi.org/10.1111/rssb.12377)。

### 置换重要性、稳健性与公平性

- 置换重要性优先在验证/测试数据计算；相关特征可能互相替代而低估单项重要性，可补充分组置换。
- 比较不同种子、折、超参数、特征集、缺失处理和解释样本下的重要性排名与曲线稳定性。
- 按教育层级、性别、地区、资源条件或其他相关群体报告性能、校准和误差；同时报告样本量和不确定性，避免小群体排名过度解释。
- 区分条件于固定切分的训练扰动稳定性、跨切分稳定性、跨时间/学校可迁移性；只完成其中一层时不得概括为“解释结果已经稳定”。

## 5. 可视化交付

- **数据与设计**：样本流向图、缺失图、目标分布、特征分布和相关性/聚类图。
- **模型性能**：同切分下的指标点图或箱线图；分类增加 ROC/PR、校准图和混淆矩阵；回归增加预测—观测与残差图。
- **全局解释**：优先使用 SHAP 条形图加 beeswarm；使用可比较的横轴，不用饼图夸大细小差异，不把四舍五入的 0% 写成无作用。
- **特征效应**：ALE/依赖图同时显示样本密度和不确定性，坐标轴写明原始量尺或标准化方式。
- **局部与群体**：仅为有理论或实践意义的代表性个案绘制 waterfall；群体比较用森林图、点区间图或热图并标样本量。
- 图注写清样本、数据切分、模型、指标/解释量、误差线、标准化和预测尺度。采用色盲友好配色，避免 3D、双轴和装饰性图形。

## 6. 结果写作与边界

按“任务与切分—模型性能—解释结果—教育含义—限制”组织结果：

1. 先报告基线、候选模型和样本外性能，不以样本内拟合选择模型。
2. 再报告特征重要性、方向、非线性、交互和群体差异，并提供稳定性证据。
3. 使用“与模型预测相关”“预测贡献较高”“模型呈现非线性模式”等表述。
4. 只有研究设计支持时才使用“影响、促进、效应、机制、阈值”或提出干预建议。
5. 讨论代理指标、测量误差、选择偏差、缺失、外部效度、算法公平和部署反馈风险。

写作时还要区分三类表述：

- **模型性能**：某模型在某一未见数据方案上的误差或拟合表现；不等于教育构念已被准确测量。
- **模型归因**：模型更依赖哪些输入信息；不等于这些因素对教育结果的净作用大小。
- **教育解释**：结合理论提出的条件化解释；若没有时间、过程、交互或干预证据，不得写成已识别机制、阶段、链条或闭环。

行动建议还需单列其证据来源：只有存在干预、过程或可执行的设计证据时，才能提出条件化行动；仅有预测性能或 SHAP 时，最多提出待验证的关注线索。

摘要只保留能回答研究问题的最小方法信息。完整算法清单、超参数、所有解释器和诊断步骤放入方法或附录，不用方法堆叠制造严谨感。

## 7. 诊断标签

按需使用 `Target Leakage`、`Criterion Contamination`、`Split Leakage`、`Preprocessing Leakage`、`Tuning Leakage`、`Test-set Selection`、`Temporal Leakage`、`Group Leakage`、`Metric Mismatch`、`Class Imbalance Gap`、`Overfitting Risk`、`Calibration Gap`、`Compositional Dependence`、`Correlated Attribution Risk`、`Hierarchy Size Bias`、`Sparse-region Interpretation`、`Subgroup Instability`、`Ablation-Fidelity Confusion`、`Explanation-Causality Confusion` 或 `Reproducibility Gap`。每项必须指向具体数据步骤、代码、表图或论文表述，并给出修正方案。

## 8. 方法参考案例

钱佳、崔晓楠（2026）的研究比较 OLS、LASSO、岭回归、随机森林、GBRT 与 XGBoost，并以样本外指标选模，随后使用 SHAP 排序预测重要性、使用 ALE 描绘非线性预测模式，可作为中文教育研究的流程案例：[DOI](https://doi.org/10.16382/j.cnki.1000-5560.2026.07.005)。借鉴其方法组合时，仍须根据当前数据补充切分策略、折内预处理、调参过程、不确定性、相关特征归因与复现信息，不照搬样本、指标或结论措辞。
