# 教育技术学可解释机器学习：论文写作与方法审计分离

用于惩罚回归、随机森林、梯度提升、XGBoost、SHAP、ALE、置换重要性或其他预测解释任务。机器学习在教育技术学论文中应服务于教育问题、构念解释和实践判断，而不是接管论文的叙述方式。

## 一、先确定任务

| 用户任务 | 优先做什么 | 不要自动做什么 |
| --- | --- | --- |
| “帮我写摘要、结果、讨论或结论” | 将结果翻译为教育问题、构念、过程结构和研究贡献 | 生成训练/测试划分、泄漏、外部样本和因果免责声明清单 |
| “帮我设计或复核模型” | 检查数据生成、切分、选模、解释和复现 | 直接产出看似验证过的研究结论 |
| “投稿前方法预审” | 形成独立的方法审计报告，定位主张受影响处 | 把审计报告复制进论文局限或摘要 |

预测研究可以回答：不同教育过程信息如何共同关联于结果、哪些信息具有更高预测重要性、关系呈现何种非线性，以及这些发现如何与已有理论对话。它也可以形成有理论依据的“过程结构”“形成逻辑”或“解释性综合”。只有在作者把这些表述当作净因果、确定时序或干预效果时，才需要进入更严格的识别审计。

## 二、论文写作轨：让模型结果成为教育发现

### 1. 方法部分

按“为什么需要建模—使用何种材料—如何比较和解释—生成什么证据”叙述。方法是研究设计的一部分，不写成软件说明书；完整参数、额外模型和重复检查可移入表格、附录或补充材料。

### 2. 结果部分

按教育问题而不是算法顺序组织：

1. 先用少量文字交代模型表现是否足以进入解释；
2. 将变量排序归并为构念、维度或过程信息；
3. 使用SHAP、ALE或消融图表说明关键发现的方向、差异或非线性形态；
4. 每个小节以一个最小经验判断收束，不在正文重复“模型只是模型”。

SHAP可写作“模型识别中更为关键的过程信息”，ALE可写作“关系在不同观测区间呈现的平台、转折或增益趋缓”。图表的技术定义、背景数据、分箱、标准化和参数，放在方法或图注。

结果的证据顺序由研究问题决定，常见但非强制的安排是：先简要说明为什么可以进入解释，再报告构念/维度层面的差异，随后报告关系形态、情境差异或稳定性证据。不要将“模型比较—SHAP—ALE—Bootstrap/消融”直接当作正文小节标题；每一层都应先回答它对教育问题增加了什么证据。

### 3. 讨论与结论

以“中心发现—教育理论或文献—过程解释—实践含义”推进。允许把问题认知要求、学生应答、思维表达、反馈调节等发现综合为课堂过程结构；讨论的任务正是把统计或模型结果转译为教育学认识。此类综合至少应让读者看见构念来源、实证支撑和相较既有研究新增的解释；没有时序、交互或干预证据时，使用“过程结构/关联配置/解释性综合”，而不把它命名为已验证的因果机制。

不要把讨论写成下列两种极端：

- 只复述“模型更依赖什么”，没有解释它对教学或学习理解的推进；
- 只列出“不能说明什么”，让每个结论都失去教育学判断。

局限如需写入论文，只保留一至两项与该研究最相关、会改变适用范围的条件，例如观察工具的覆盖范围或研究情境。不要自动附加“还需外部样本、纵向数据、干预研究、嵌套交叉验证”等完整愿望清单。

### 4. 适合论文的表达选择

| 发现类型 | 可采用的学术表达 |
| --- | --- |
| 多类过程信息共同出现 | 呈现出……的过程结构；共同构成……的课堂图景；显示……之间的关联配置 |
| 重要性与方向 | 其中……尤为关键；……是区分……的重要过程信息；……与较高/较低……相联系 |
| 曲线形态 | 呈现非线性关系、平台特征、增益趋缓或区间差异 |
| 教学含义 | 促使课堂观察从……转向……；为……提供分析视角、评价维度或改进方向 |

这些不是句式模板。根据论文的理论、数据和文类自然改写，不仿写任何作者。

## 三、方法审计轨：只在用户明确请求时开启

检查以下事项，并单列“事实—影响—修复”而非写入论文正文：

1. 目标变量与特征是否重叠，是否可能重建同一评分规则；
2. 分析单位、群组或时间边界是否与使用情境相符；
3. 插补、标准化、编码、降维和特征选择是否在相应训练数据内完成；
4. 模型比较、调参与最终性能报告是否使用了相称的验证设计；
5. 重要性、曲线、节点删除和稳定性检查分别回答什么问题；
6. 结果表图、代码、数据版本和研究叙述能否相互追溯。

审计中发现同一测试集参与选模、泄漏或不可复核结果时，优先限制与之直接相连的“独立最终最优性能”“严格阈值”“确定因果效果”等承诺。除非这些承诺正是论文中心结论，不将问题扩散为全文降格、摘要免责声明或结论中的长段自我否定。

## 四、图表

- 性能比较：一张紧凑表或点图即可；正文只提解释模型的选择理由。
- 重要性：优先展示能支持构念归并的层级或分组图，而不是全部变量的视觉排名。
- 非线性：ALE/依赖图服务于一个教育问题；图题与图注说明指标和取值范围，正文讨论其教育含义。
- 稳定性与消融：仅在它们真正支撑文章中心发现时呈现；否则放入附录或补充材料，不让“方法证明”淹没教育发现。

## 五、必要的技术依据

设计或审计时可核对 [scikit-learn 数据泄漏与 Pipeline 指南](https://scikit-learn.org/stable/common_pitfalls.html)、[模型评估](https://scikit-learn.org/stable/modules/cross_validation.html)、[SHAP TreeExplainer](https://shap.readthedocs.io/en/latest/generated/shap.TreeExplainer.html) 与 [ALE 方法论文](https://doi.org/10.1111/rssb.12377)。这些资料用于改善分析和复现，不规定中文教育技术学论文的摘要或讨论必须采用技术审计语言。

钱佳、崔晓楠（2026）以机器学习讨论创新能力的多类影响因素，并将模型发现组织为发展阶段、因素结构和培养启示，可作为“教育问题—发现簇—教育含义”组织方式的参考，不复制其结论或句式：[DOI](https://doi.org/10.16382/j.cnki.1000-5560.2026.07.005)。
