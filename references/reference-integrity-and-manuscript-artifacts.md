# 参考文献完整性与稿件工件质检

用于参考文献系统审计、匿名稿、图表、补充材料和 DOCX/PDF 交付。具体样式始终服从目标期刊当前官方指南；GB/T 7714、作者—年份制或其他格式不得凭领域惯例推定。

## 一、参考文献系统审计

逐项检查：

1. 正文每个引文均有文后条目，文后每个条目在正文中承担明确功能。
2. 编号制按目标期刊规则保持顺序、区间与标点一致；作者—年份制核对姓名、年份和同年后缀。
3. 同一来源未重复建项，替换或删除文献后已同步更新正文、图表、脚注和附录。
4. 题名、作者、年份、期刊/出版社、卷期、页码与持久定位信息可由原始页面核验；持久定位信息优先 DOI，缺少 DOI 时使用官方 URL、数据库记录号或出版社页面，不能为凑齐字段而编造 DOI。
5. 直接引语核对原文、页码与上下文；二手转引明确标识。
6. 网络资源记录真实访问日期，不猜测发布日期；失效链接寻找官方归档或替代稳定入口。

优先复核高风险条目：仅凭记忆补全、只见二手引文、元数据不一致、近期未正式出版、预印本与正式版混用、作者同名、自动导入字段异常、网页链接失效。核验失败时标记 `[待核验]`，由作者决定删除、替换或补证，不设置武断的搜索时限。每条拟引用来源至少能形成“题名—作者—年份—载体—定位信息”对应；任一关键字段冲突时，不将其用作关键论证依据。

参考文献数量、语种、年代和文献类型没有跨期刊通用比例。只检查它们是否足以覆盖问题、理论、方法、数据与争论，并核验目标期刊的明文要求。

## 二、投稿包清单

根据期刊要求组装并互相核对：

- 主稿、匿名稿、标题页/作者信息页。
- 中文与英文题目、摘要、关键词。
- 图表原文件、附录、补充材料、数据/代码可用性声明。
- 伦理审批、知情同意、利益冲突、基金和作者贡献声明。
- 投稿信、审稿回应函及期刊要求的其他表单。

匿名化不只删除姓名：还需检查文件属性、修订记录、批注、基金、致谢、自引措辞、图片元数据、共享链接和可识别项目描述。作者账户、身份信息、协议签署和最终投稿操作由作者本人完成。

## 三、稿件工件工作流

对于 DOCX/PDF 或图表交付，采用：

```text
确认目标模板 → 生成/修改源文件 → 应用样式 → 导出目标格式
             → 渲染检查 → 内容与视觉复核 → 形成提交版本
```

不得只检查可编辑文本而跳过最终渲染。视觉复核至少覆盖：分页与标题孤行、图表裁切、字体替换、公式、脚注、交叉引用、目录、页眉页脚、匿名信息、乱码和空白页。

图表须与正文数字、样本、变量、单位和图注一致；图像不能单独传递正文未支持的因果、方向或等级判断。若用户提供期刊模板，优先在模板样式上操作，不重新猜测版式。

## 四、作者责任与原创性

不得把编辑目标设为规避 AI 检测。应检查论证是否来自真实研究过程、作者是否能解释关键决定、引用和数据是否可追溯、自动生成内容是否经过实质核验，以及期刊或机构是否要求披露工具使用。

保留必要的检索记录、分析代码、版本历史、修订台账和作者核验记录。若 AI 参与过检索、翻译、摘要、改写、代码或图表草拟，在项目内部记录使用环节、人工核验动作、使用的材料版本和责任人；是否对外披露以目标期刊或机构当期规则为准。模板化表达可因其妨碍准确性而修改，但不能故意插入错误、异常标点或低质量表达来影响检测工具。

直接使用 `assets/submission-package-checklist.md` 完成最终交付核对。
