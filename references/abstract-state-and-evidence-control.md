# 摘要的证据状态与信息密度控制

用于预测研究、统计实证、设计实践、质性和理论论文的摘要诊断与改写。它不规定某一本期刊的句式，也不把“更像教育学”理解为删去研究设计或把预测关联改写成教育机制。具体字数、关键词和结构式摘要要求仍须以目标期刊当期官方页面核验。

## 一、先区分三种产物

“摘要”既是论文中的一段文字，也是对研究证据的公开承诺。证据问题未修复时，直接要求生成“投稿摘要”会逼迫模型在两种坏结果之间选择：要么把内部风险和方法清单塞进摘要，要么省略限定、把当前结果包装成成熟结论。因此先确定产物类型。

| 产物 | 适用状态 | 交付方式 | 不可做什么 |
| --- | --- | --- | --- |
| 诊断与摘要论证卡 | `阻断 / 探索性可写 / 投稿就绪` | 说明主张、证据、缺口和修复动作；不进入论文 | 用诊断表替代摘要正文 |
| 当前版本摘要 | `阻断 / 探索性可写`，且用户要求改写 | 直接交付连续、自然的当前版本文本；只有用户询问投稿判断，或风险会改变文中关键主张时，才在稿外集中说明待修复项 | 把未确认性能写成独立最终验证，或把预测关联写成已证实因果效应 |
| 投稿摘要 | 仅 `投稿就绪` | 交付不含内部标签的连续摘要 | 隐藏未修复的选模、泄漏、测量或结果问题 |

`阻断` 不是拒绝协助写作。它只限制尚未成立的论文承诺，不限制围绕已有材料进行自然、可读的改写。不要默认在每一版摘要外加“不可投稿”标签；只有用户明确要求投稿就绪判断、或该标签对避免误用不可缺少时才加。

## 二、摘要不是固定的“四发现模板”

非结构式中文摘要通常需要完成五项信息移动：具体问题、材料/设计、关键发现、由发现直接支持的认识、必要边界。移动顺序可以为读者理解服务，但不强制“四个发现”“先算法后教育含义”或“理论—方法—实践”三格贡献。

推荐从一个中心回答组织，再保留一至数项真正改变解释的次级结果：

1. **问题句**：指出对象、活动、结果或决策中的具体未解问题，而不是只说“重要”。背景事实若无法由正文来源支持，不以宏大价值判断填补。
2. **设计锚点**：交代分析单位、材料规模或来源、研究类型，以及读者判断结论强度所必需的验证角色。预测研究不能为了“少一点计算机语言”而删去全部验证信息。
3. **发现簇**：先报告最能回答问题的一项结果；性能、模型归因、曲线形态和稳定性只保留改变解释所必需的部分。不能因为有四类分析就必须写成四条发现。
4. **认识或用途**：把结果翻译为可观察的诊断优先级、待检验解释或适用条件；不能把变量并列、模型排序或系统功能改写为过程机制、干预效果或评价处方。
5. **边界安排**：当摘要本身使用了可能造成误读的强动词，或读者无法从研究设计判断结论强度时，补一处具体限定。若全文已用相称动词，且局限将在讨论中交代，不必为横断面、同源观察或预测研究机械添加“并不构成因果证据”式句子。

## 三、可解释机器学习摘要的四层词义

同一段摘要可出现模型性能、模型归因、教育解释和行动含义，但四层不能互换。

| 层次 | 可以在证据充分时写什么 | 不能升级成什么 |
| --- | --- | --- |
| 模型性能 | 某锁定模型在独立最终测试或说明清楚的验证方案上的误差、R²、校准或不确定性 | 教育构念已被准确测量，或模型因分数较高而“最佳” |
| 模型归因 | 模型更依赖哪些输入；归一化 SHAP 份额在何一分母下计算 | 变量对结果的净作用、影响大小或因果贡献 |
| 预测函数形态 | 在密度、分箱和重复拟合支持下，模型预测曲线呈现的平台、非线性或局部变化 | 可直接执行的教学阈值、真实世界剂量效应 |
| 教育解释与行动 | 与理论一致的诊断线索、待检验解释、条件化的课堂关注点，或明确标为解释性综合的研究认识 | 把模型归因直接当作变量的净效应，或把未经时序/过程检验的解释性综合写成实证已证实的因果机制 |

对当前留出集参与选模的稿件，不能把性能写成“独立最终测试已确认的最优表现”。但可以用“在当前划分下表现较好”“候选模型中预测表现突出”等准确说法，并在需要投稿判断时说明独立确认的必要性。删除某组输入后性能下降，只能说明模型对该输入组的依赖，不能推出移除相应教学行为会降低成效。

## 四、对教育问题的解释不等于越界因果化

把摘要从算法报告改成教育研究叙述，正确做法是说明结果回答了哪个教育问题、对课堂观察或学习支持提供了什么认识。要审查的是一句话是否把模型归因改写成已验证效应，而不是机械删除“影响”“机制”“形成”或“共同”等词。研究问题、文献讨论、理论解释和局限都可以使用这些词；实证结论则要让动词与证据层次相称。

下列替换需回查证据，而不是机械润色：

| 容易出现的写法 | 需要的证据 | 在预测研究中的保守表述 |
| --- | --- | --- |
| “A、B 共同塑造高质量课堂” | 时序、交互、过程或干预证据 | 若作为实证结论，改为“课堂过程信息在模型中共同呈现出与教学成效相关的预测模式”；若作为理论解释，明确是“据此形成的待检验解释” |
| “A 影响教学成效” | 相称的因果识别 | “A 与模型对成效的预测相关” |
| “出现阈值效应” | 局部样本密度、不确定性与跨切分一致性 | “当前预测曲线在观测区间内出现转折/平台候选” |
| “GBRT 最优，R² 为……” | 验证集选模后的一次独立最终测试，且比较不确定性已报告 | “当前验证方案下，锁定模型的表现为……” |
| “为教师提供改进处方” | 干预、实施或过程证据 | “为课堂观察和后续干预设计提供优先核查线索” |

### 语言不是禁词表

按三个层次判断，而不是见词即删：

1. **实证结论**：准确说明数据直接支持的关系、预测或差异，不把模型依赖当作净效应。
2. **理论解释**：允许在理论和已有研究支撑下提出“这提示”“可从……理解”“可能的形成线索”等综合解释；应与实证结论相邻，但不冒充已经验证的机制。
3. **问题与意义**：可以自然讨论“高质量课堂何以形成”“学习者创新能力的影响因素”等领域问题，不把问题措辞误当作研究已证明的结论。

当用户提供的原句在强度上有争议时，先保留其研究意图，说明争议发生在“实证结论”还是“理论解释”，再给一版相称改写；不要在未解释的情况下把整段改成免责声明。

## 五、摘要改写的最小检查

1. 在内部判断发布状态与产物类型，再检查题目、摘要、结果和结论是否使用同一强度；只有投稿判断或当前风险会改变文本时，才把状态说明展示给用户。
2. 删除算法目录式复述，但保留研究类型、材料、分析单位和验证角色中读者理解所必需的部分。
3. 对每一项数字标注其角色：性能、归因、曲线形态、稳定性或描述统计；没有改变主回答的数字移至正文表图。
4. 检查教育解释是否引入了正文没有的时序、交互、机制、因果或推广承诺。
5. 只有目标期刊官方明确要求时才加结构式标签；没有官方依据时，不凭“某刊风格”强行套用发现数量、句式、算法删减规则或免责声明。

## 六、删减检验：控制信息带宽，不牺牲研究判断

摘要不是整篇实证研究的“压缩说明书”。方法透明性主要由方法、表图、附录和可复现材料承担；摘要只承担让读者迅速判断问题、研究对象、中心发现和含义的任务。但摘要也不能把设计信息删到无法辨认结论类型。

按两步处理：

1. **先删**：删去按代码顺序出现的模型清单、解释器清单、重复性能指标、归因比例、稳定性系数和消融结果，保留问题、材料/设计、中心发现、含义和边界。
2. **按证据放回**：逐项判断被删信息是否改变读者对研究类型、验证角色、中心发现可信度或适用范围的判断。只有会改变判断的信息才回到摘要；通常每一证据层保留一个最有解释力的锚点，不把同一层的多个技术细节并列展示。

因此，“R²、SHAP 百分比、Bootstrap 系数、删除实验都应放回正文”只能是对多数常规教育实证的压缩建议，不能成为不分研究类型的禁令。若性能比较或方法创新正是研究问题，摘要可保留相称的模型或指标；若它们只是支撑教育发现的技术过程，应把空间留给中心发现与有限含义。
