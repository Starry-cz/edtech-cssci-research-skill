# 验证场景矩阵

更新 Skill 后，用下表的代表性任务检查路由是否正确。验证关注“是否给出可执行、可核验的研究支持”，不以生成看似完整的论文段落作为通过标准。

| 触发任务 | 预期资源或模式 | 核心输出 | 禁止行为 |
| --- | --- | --- | --- |
| “AI 教育很热，帮我定题” | `topic_diagnosis`；选题—问题文件与选题画布 | 七道质量门、对象、情境、机制、证据需求与候选问题 | 把政策口号或技术名称当作研究问题 |
| “国际报告都在谈 AI，如何说明研究意义” | 外部证据与研究定位文件；`topic_diagnosis` | 问题情境、待检验命题、治理条件与本研究证据需求 | 把报告判断、期刊范围或演讲观点写成本研究发现 |
| “围绕 AI 教育写一篇论文，主线怎么搭” | `argumentation_blueprint`；论证与修订工作底稿 | 问题—回答—证据—贡献—边界主张图与待补材料 | 用修辞性创新点或泛化效果填补论证空白 |
| “把这个热点改成研究问题” | `research_question_refinement`；选题画布 | 主问题、相互依赖子问题、概念边界、证据与不可回答部分 | 用宽泛变量关系替代研究问题 |
| “生成式 AI 如何赋能大学教学？” | `topic_diagnosis`、`research_question_refinement` | 将口号拆为主体、活动、技术角色、机制、证据、风险与边界，并给出问题类型匹配 | 直接产出“现状—问题—对策”或无比较/过程的因果题 |
| “帮我列论文提纲” | `outline_building`；章节收束文件 | 章节功能、证据接口与论证顺序 | 只按常见章节名机械罗列 |
| “我的全文主线和贡献不清楚” | `argumentation_blueprint`；论证与修订工作底稿 | 问题—回答—证据—贡献—边界主张图 | 以修辞性创新点替代可支持的贡献 |

| “设计一个智慧课堂/平台模型” | `practice_model_design`；模型与系统设计文件 | 设计原则、活动链、实施条件和过程/结果证据 | 把功能或上线写成教育效果 |
| “文献很多，综述写不出来” | 两个文献模式；综述矩阵 | 检索路径、主题矩阵、争议和缺口 | 编造出处或作者罗列 |
| “核对这条引文能否支撑主张” | `citation_and_evidence_check` | 主张—来源映射与核验状态 | 未读原文即确认其结论 |

| “图上有星号，能否直接写教学显著提升” | `education_data_analysis`；统计报告与图表证据门 | 先识别独立单位、比较家族、效应量/区间和图注口径，再判断表述强度 | 用星号、软件名称或显著性替代效应大小、设计与教育意义 |
| “我想预测并用 SHAP 解释” | `interpretable_ml_analysis`；可解释机器学习文件 | 泄漏控制、切分、模型比较、解释与稳定性 | 将预测或特征重要性写成因果机制 |
| “层次分类重跑后如何报告” | 可解释机器学习模式；层次分类文件 | 层级标签规则、切分、误差追踪与可视化 | 在层级间复用泄漏信息或只报最好指标 |
| “为我的研究框架画一张图” | `framework_defense`；跨 Skill 工件路由；当前可用的 `drawio` Skill | 图表说明书、节点/箭头证据状态、可编辑图交付要求与图文一致性检查 | 未经理论或材料支持就补造机制、因果箭头、闭环或中介 |

| “精读这篇 PDF，并告诉我它如何支撑我的综述” | `literature_search_to_review`；溯源式论文阅读 | 阅读范围、材料状态、可定位笔记、图表功能、术语表与可引用边界 | 只读摘要就概括全文方法、图表、结果或局限 |
| “帮我写摘要/引言/讨论” | `section_revision`；写作工作流 | 章节承担的判断、所需证据、修复方案和边界 | 将摘要承诺、结果、讨论和贡献混写 |
| “摘要写成‘目的：方法：结果：结论：’，帮我改成 C 刊表达” | `section_revision`；投稿正文表层控制与审计模板 | 先核验期刊是否要求结构式摘要，再输出连贯清洁摘要与分离的修改说明 | 把构思维度、风险标签、占位符或三格贡献模板原样留在正文 |

## 统一通过条件

- 输出能明确本次任务的主模式、材料状态、核心判断、证据来源和结论边界。
- 文献、数据、期刊要求和审稿意见均只基于已给材料或已核验来源；未知项保留为待核验。
- 数据分析包含过程审计、验证或稳健性判断；模型/实践研究包含设计理由和相称的证据类型。
- 写作或返修任务先处理主张、理论、设计和证据，再处理段落与措辞；一项结构性修改会检查摘要、图表、附录、回应函等连锁位置。
- 投稿清洁稿不出现构思标签、模式名、诊断标签和占位符；结构式摘要、编号研究问题和固定栏目只按目标期刊官方要求启用。
- 涉及摘要、题目、主要结论和投稿判断时，必须先给发布状态；`阻断` 不得输出定稿式摘要、结论或提交建议。
- 关键结论、图表与实践建议均须建立“主张—决定性证据—比较/反例—稳健性—边界”验证包；缺失决定性证据时不能用方法堆叠、星号或图形修辞放行。
- 图表、文档和数据表等工件先说明服务判断和证据来源；仅调用当前环境中可用的专项 Skill，并在交付前回检图文、数值与结论边界。
