# 代表性验证场景

用于更新 skill 后的前向测试。测试输出不必相同，但必须满足预期行为。

1. **宽泛选题**：“人工智能赋能教育”。应选择 `topic_diagnosis`，标记主题总结/问题过宽，给出聚焦方向且不编文献。
2. **作者罗列综述**。应标记 `Literature Listing`，按争论、机制、方法或场景重组并指出缺失文献簇。
3. **理论名称堆叠**。应标记 `Theory Decoration`，要求概念与学习活动、变量/编码和结果解释连接。
4. **不明来源学习日志**。应标记 `Data Provenance Gap`，询问事件生成、分析单位、时间窗口、缺失和权限。
5. **横断面问卷写因果**。应标记 `Causal Overclaim`，给安全关联表述和可加强识别的设计。
6. **检索并导入 Zotero**。应区分来源/阅读/导入状态，先建检索日志和结构化笔记，不把导入等同于已读。
7. **全文诊断**。应逐项使用 `Pass / Partial / Fail`，每个判断引用用户材料并给具体操作。
8. **可解释机器学习**。给定学生平台日志与学习结果，应按学习者/班级/学校或时间切分，比较基线与候选模型，报告样本外性能和不确定性，再用 SHAP/ALE 解释；不得把重要性写成因果效应。
8. **章节改写**。应保留可核验主张，标记缺失证据，不能为了流畅补造数据或引文。

失败信号：只给“加强论证”等空话；未选择模式；无材料证据却评分；虚构来源/数据；把显著性或预测写成因果；先用全数据插补/标准化/降维再切分；用测试集调参；把 SHAP 重要性称为效应量；把横截面学段差异称为长期效应；把 Zotero 记录当全文已读；把公共管理示例直接带入教育技术学任务。
