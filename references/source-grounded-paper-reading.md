# 溯源式论文阅读与笔记交付

用于精读论文、做中英文对照笔记、为综述提取方法/证据、解读图表或把用户提供论文转成可核验阅读材料。它不等于“看见摘要就总结全文”，也不要求复制或再发布论文全文。

## 一、先识别材料状态

| 状态 | 允许产出 | 不允许产出 |
| --- | --- | --- |
| 全文可读 PDF/HTML | 结构化阅读笔记、图表说明、方法/结果/局限提取与定位。 | 超过合理需要的大段逐字复制。 |
| 扫描件或 OCR 不完整 | 标出页码、低置信段落、图表缺失，输出可复核的部分。 | 把 OCR 猜测写成原文事实。 |
| 只有 DOI、题录或摘要 | 题录核验、摘要范围内概括、获取全文的下一步。 | 声称已读方法、图表、结果或局限。 |
| 用户粘贴节选 | 节选级解释、术语表和待补上下文。 | 推断全文论证或期刊格式。 |

## 二、源图谱优先于总结

阅读前建立“源图谱”：论文题录/版本、页码或章节、图表编号、文本可读性、缺失页、术语和用户关心的问题。阅读时每个关键笔记至少标明来源位置（页码、章节、表图或段首短语），以便用户回到原文核验。

对全文阅读，按以下顺序记录：研究问题—对象/材料—理论/概念—设计与数据—主要发现—图表功能—作者的解释—作者承认的局限—本研究可如何使用。区分“作者报告”“阅读者解释”和“尚待核验”。

## 三、图文对应与术语一致

- 图表只在能够读取图题、图注和相关正文时解释；看不到原始数据时不反推精确数值。
- 建立术语表，统一中英文构念、变量、模型与方法名；翻译优先保持研究含义和不确定性，不追求逐词替换。
- 需要双语对照时，以段落或小节为单位，保留来源位置；未提供授权时不输出大篇幅全文翻译，应改为分段解释或用户指定范围的翻译。

## 四、交付格式

默认输出：

1. 阅读范围与材料状态；
2. 溯源笔记表（位置—原文内容类型—准确概括—可用于何种主张—限制）；
3. 图表清单（图表回答的问题、可读证据、不能推断的内容）；
4. 术语表；
5. 与当前研究的关系和待核验项。

将阅读状态同步回 `literature-search-and-zotero.md` 的 `metadata only / abstract only / full text read` 体系。需要将笔记用于引用时，再读取 `evidence-and-citation.md`。
