# 教育技术学 C 刊终稿自检

只用于正文基本完成、准备投稿、送导师或同行前的严格审查；提纲阶段不要调用。

## 默认输出

1. 总体判断：接近可投稿、需要中修、需要重构。
2. 主要问题：按严重程度列 5—8 项，定位到具体章节或表达。
3. 修改建议：给出可执行方向，必要时提供短示例。
4. 反 AI 痕迹快扫：标出模板句、空泛句、对偶句、路标句和宣传腔。

## 审查维度

### 问题意识与对象

检查题目核心概念是否贯穿全文，引言是否把政策、热词或技术更新当成研究问题，研究对象与场景是否不可替代，技术问题是否最终落到学习、教学、评价或教育组织问题。

### 文献与概念

检查综述是否呈现研究推进而非作者罗列；概念是否从定义进入变量、条件、编码、活动或观察指标；研究缺口是否具体且能由本文处理。

### 理论与机制

检查理论是否进入变量选择、设计原则、案例编码、指标构建或结果解释。机制不得停在“技术赋能、资源整合、教学创新”，应连接主体、活动、技术功能、规则、过程证据与结果。

### 方法设计

- 问卷/结构方程：量表来源与情境适配、样本结构、共同方法偏差、信效度、测量与结构逻辑。
- 实验/准实验：分配方式、对照条件、干预内容、实施忠实度、测量时点、效应量、污染与剩余偏差。
- 案例/质性：案例选择、研究者位置、过程证据、访谈与观察、编码、反例和替代解释。
- 学习分析：日志生成、分析单位、清洗与特征、验证划分、教育含义、隐私、公平和算法边界。
- 设计型研究：真实问题、理论驱动设计、迭代证据、修改理由和设计原则的跨轮支撑。
- QCA：条件选择、校准依据、案例数量、必要性/充分性、一致性/覆盖度与反事实解释。
- 政策/文本分析：语料来源、筛选、时间边界、编码规则、概念提取和可复核性。
- 系统综述：问题与协议、数据库、完整检索式、纳排标准、筛选流程、质量评价和综合方法。
- 混合方法：采用两类数据的必要性、设计类型、整合节点、冲突处理和整合后的新增认识。

### 证据、结果与贡献

检查每项关键判断是否有文献、数据、日志、课堂材料、访谈、案例或政策支撑；删除“普遍存在”“师生反映强烈”“各地积极探索”等无证据表达。结果不能只列图表，应从总体到差异再到机制。结论不得超出样本、设计和情境，贡献与建议必须来自发现。

## 反模板快扫与通过标准

修改机械对偶、高频路标、宣传腔、作者排队、案例点缀、机制标签和“一是二是三是”式复述。通过标准：题目关键词持续回应；问题明确；综述推进；概念可分析；理论进入设计与解释；方法可回答问题；关键判断有证据；结果有层次；结论克制；动态期刊要求已核验。
