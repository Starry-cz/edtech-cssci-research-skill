# 教育技术学 C 刊终稿自检

只用于正文基本完成、准备投稿、送导师或同行前的严格审查；提纲阶段不要调用。

## 默认输出

1. 发布状态：`阻断 / 探索性可写 / 投稿就绪`，并说明当前允许的交付。
2. 主要问题：按严重程度列 5—8 项，定位到具体章节或表达。
3. 修改建议：给出可执行方向，必要时提供短示例。
4. 原创性与表达快扫：标出模板句、空泛句、对偶句、路标句和宣传腔，并检查关键判断是否体现真实研究过程与作者核验。

若用户同时要求改写，分开交付“清洁正文”和“诊断/修改说明”。`Pass/Partial/Fail`、风险标签、缺失信息和占位符只进入修改说明，不得混入清洁正文。

每个适用标准使用 `Pass / Partial / Fail`：`Pass` 表示材料完整且相互一致；`Partial` 表示已有基础但缺关键说明或证据；`Fail` 表示缺失、矛盾或构成投稿风险。引用论文中的具体位置或原句，不允许只给抽象评分。材料未提供时标记 `[needs user verification]`。

## 审查维度

终稿采用四类互补视角，不虚构“专家结论”：领域视角检查教育对象、技术与情境事实；理论视角检查概念和机制；方法视角检查数据与识别；编辑视角检查问题价值、结构、可读性和目标期刊适配。无法由材料确认的事项标记 `[needs user verification]`。

### 问题意识与对象

检查题目核心概念是否贯穿全文，引言是否把政策、热词或技术更新当成研究问题，研究对象与场景是否不可替代，技术问题是否最终落到学习、教学、评价或教育组织问题。

同时抽取论文的主论证链：真实问题、研究问题、证据、最小贡献和结论边界。题目、摘要、引言、方法、结果和结论中任一节点不能对应时，分别标记 `Abstract Drift`、`Structure Drift`、`Evidence Gap`、`Contribution Inflation` 或 `Conclusion Overreach`，并先修复会传播到全文的节点。

### 文献与概念

检查综述是否呈现研究推进而非作者罗列；概念是否从定义进入变量、条件、编码、活动或观察指标；研究缺口是否具体且能由本文处理。

### 理论与机制

检查理论是否进入变量选择、设计原则、案例编码、指标构建或结果解释。机制不得停在“技术赋能、资源整合、教学创新”，应连接主体、活动、技术功能、规则、过程证据与结果。

涉及模式或框架时，逐项核对每个层、环、箭头和标签的理论来源、观测证据、操作过程和应用主体。只有命名或绘图、没有可解释关系的，标记 `Mechanism Gap`；案例只作展示而未验证任何命题的，标记 `Evidence Gap`。

### 方法设计

涉及样本、测量、日志、清理、模型与因果解释时，同时读取 `data-collection-and-analysis.md`，不得使用不考虑设计条件的固定样本量或单一统计阈值。

- 教育数据分析：检查数据字典、原始与分析数据边界、排除与缺失机制、描述统计、可视化、量表/指标构念、模型选择、假设诊断、效应量与不确定性、稳健性、复现和教育解释；详细流程读取 `education-data-analysis.md`。

- 问卷/结构方程：量表来源与情境适配、样本结构、共同方法偏差、信效度、测量与结构逻辑。
- 实验/准实验：分配方式、对照条件、干预内容、实施忠实度、测量时点、效应量、污染与剩余偏差。
- 案例/质性：案例选择、研究者位置、过程证据、访谈与观察、编码、反例和替代解释。
- 学习分析：日志生成、分析单位、清洗与特征、验证划分、教育含义、隐私、公平和算法边界。
- 可解释机器学习：目标与特征是否构念重叠；切分是否早于预处理；多模型选模是否隔离最终测试集；SHAP/ALE是否报告假设、不确定性、相关/组成特征和因果边界；维度汇总是否处理成员数量偏差；小样本群体是否停止排名；删除节点是否被准确称为消融而非保真度。

以下任一项未解决时直接判为 `submission blocker`，对应发布状态为 `阻断`：目标与特征存在直接计分重叠；同一教师/班级/学校跨训练测试；预处理在切分前拟合；同一测试集承担多模型选模与最终评价；主要结果无法从分析文件复核。允许保留探索性分析，但题目、摘要和结论必须同步降级；不得生成看似定稿的摘要或提交判断。
- 设计型研究：真实问题、理论驱动设计、迭代证据、修改理由和设计原则的跨轮支撑。
- QCA：条件选择、校准依据、案例数量、必要性/充分性、一致性/覆盖度与反事实解释。
- 政策/文本分析：语料来源、筛选、时间边界、编码规则、概念提取和可复核性。
- 系统综述：问题与协议、数据库、完整检索式、纳排标准、筛选流程、质量评价和综合方法。
- 混合方法：采用两类数据的必要性、设计类型、整合节点、冲突处理和整合后的新增认识。
- 概念/实践模型：主难题是否证据化；目标、构念、数据、交互、服务与评价是否闭合；提出、实现、可用、有效与推广是否被区分。
- 国际政策比较：共同维度、案例情境、共性/差异、本土转译和实施效果边界。
- 画像/测评/推荐：标签的构念依据、观测点、数据来源、算法误差、人工复核、反馈行动与申诉纠错。

### 证据、结果与贡献

检查每项关键判断是否有文献、数据、日志、课堂材料、访谈、案例或政策支撑；删除“普遍存在”“师生反映强烈”“各地积极探索”等无证据表达。结果不能只列图表，应从总体模式到关键差异再到最小判断；只有存在过程、时序、交互或因果证据时才进一步讨论机制。结论不得超出样本、设计和情境，贡献与建议必须来自发现。

逐条区分贡献强度：框架或概念只能说明“提出/界定”，描述性材料只能“描绘/识别”，相关或预测只能报告“关联/预测”，设计原则需要跨轮实施与过程证据。系统上线、案例展示、政策目标、显著性或 SHAP 重要性均不自动构成教育效果或因果机制。

检查结果、讨论与结论是否分工：结果报告观察到什么，讨论解释其与理论、设计和替代解释的关系，结论限定能够主张什么。三者混写时，优先重排功能而非改写措辞。

预测研究额外检查：题目中的“塑造、促进、何以达成、机制”是否超过识别强度；SHAP 是否被写成因素对教育结果的贡献；ALE 是否被写成现实阈值；多个并列特征是否被发明为过程链或闭环；消融是否被写成改变教育行为后的因果结果。

同时读取 `claim-evidence-validation-contract.md`：每项题目、摘要、结果、图表和建议中的关键主张都应找到决定性证据、比较/反例、稳健性与边界。缺失决定性证据属于 `submission blocker`；比较或稳健性尚未完成时只能进入探索性写作。

### 投稿表层与内部痕迹

同时读取 `publication-prose-and-style-control.md`，并使用 `assets/manuscript-surface-audit-template.md`。清洁稿必须满足：

- 非目标期刊明确要求时，摘要不出现“目的：方法：结果：结论：”标签；
- 不出现模式名、资源文件名、`RQ1` 强制编号、诊断标签、`Pass/Fail` 或待办占位符；
- 不机械凑齐“理论层面—方法层面—实践层面”贡献；
- “本文/本研究”按作者决策和主语清晰度适量保留，不作一刀切删除；
- 结构完整体现在论证顺序中，不依赖冒号、引号、对偶口号和对称清单展示。

若目标期刊要求结构式摘要或固定栏目，记录官方链接与核验日期后按其要求执行；不能凭通用写作模板推定。

### 图表与应用转化

检查研究路线图是否说明证据生成，机制图是否说明关系，系统图是否展示输入—处理—反馈—行动，结果图是否突出差异。图表节点须与正文、方法和结果对应。建议应能回溯到具体发现，并明确责任主体、实施条件和跟踪指标。

若论文以矩阵、连续谱、类型学或原创模型为主要贡献，同时读取 `framework-defense-and-figure-audit.md`；检查竞争维度、分析层级、缺失组合、边界案例、操作化与图文一致性。

涉及结果图、流程图或框架图时，检查是否有 `figure-evidence-contract-template.md` 对应的核心结论、面板任务、统计信息、数据/代码母版和审稿风险；图表不能只因美观或数量丰富而放行。

### 参考文献、工件与作者责任

终稿同时读取 `reference-integrity-and-manuscript-artifacts.md`，检查正文—文后条目、主稿—匿名稿、数据—图表—摘要—结论和投稿材料版本一致性。不得以规避 AI 检测为目标；应检查作者能否说明关键决定、来源和分析过程，自动生成内容是否经过实质核验，以及是否需要按期刊规则披露工具使用。

若研究问题、理论、样本、清理规则、分析模型、结果解释或结论在本轮发生变化，同时检查其对题目、摘要、综述缺口、图表、附录、数据/代码说明、回应函和投稿材料的连锁影响。只改正文而未同步这些工件时标记 `Revision Cascade Gap`。

## 原创性与反模板快扫

修改机械对偶、高频路标、宣传腔、作者排队、案例点缀、无证据机制标签和为了整齐而生成的“一是二是三是”式复述。不要把所有枚举词或“本文/本研究”视为禁词。通过标准：题目关键词持续回应；问题明确；综述推进；概念可分析；理论进入设计与解释；方法可回答问题；关键判断有证据；结果有层次；结论克制；清洁稿无内部痕迹；动态期刊要求已核验。

最终把问题分成 `submission blocker`、`major revision` 和 `polish`，分别说明定位、依据、操作和预期改善，再给出 `submit`、`revise-before-submit` 或 `restructure-before-submit` 判断。

投稿前如有本地稿件文件，可先运行 `scripts/audit_manuscript_surface.py` 做表层候选扫描；扫描结果不能替代对切分、泄漏、准则污染、复现和目标期刊规则的人工核验。
