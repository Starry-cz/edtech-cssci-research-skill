# 文献检索、结构化阅读与 Zotero 工作流

在用户要求实际检索中英文文献、整理 Zotero、生成阅读笔记、综述矩阵或从检索结果写综述时读取。该流程不能替代 `evidence-and-citation.md` 的证据纪律。

## 前置条件与工具路由

- 至少获得主题、研究问题、题目、核心概念、对象/场景、方法或时间范围中的一项。
- 中文文献优先使用可用且用户有权限的 CNKI、国家哲学社会科学文献中心、期刊官网或高校数据库工具。
- 英文文献按领域使用 ERIC、Web of Science、Scopus、Google Scholar、出版社和 DOI/Crossref 元数据工具。
- Zotero 写入仅在用户明确要求或确认后执行；无连接器、登录或数据库访问时，报告阻断并继续给检索式、筛选表和空矩阵。

## 检索到综述

1. **界定范围**：列中英文关键词、同义词、教育层级、学习/教学活动、技术、结果、方法、地区、时期和排除词。
2. **构造批次**：核心概念+对象；机制/关系+对象；方法/场景；经典/高影响与最新研究；反向结果或批判性研究。
3. **记录检索**：数据库、日期、完整检索式、过滤条件、结果量、筛选阶段和去重方式。
4. **筛选来源**：按与研究问题的直接性、原始性、权威性、方法透明度和时间相关性纳入；不能只按可下载性选择。
5. **标记状态**：`candidate source`、`metadata only`、`abstract only`、`full text read`、`in Zotero`、`imported to Zotero`。
6. **结构化阅读**：记录研究问题、概念/理论、方法/数据、主要发现、贡献、局限、可用主张、与本文关系和风险。
7. **建立矩阵**：按争论、机制、技术功能、学习活动、对象、方法或时期聚类，比较共识、分歧和边界。
8. **撰写综述**：只使用已核验笔记支持具体主张，按文献簇推进，最后形成 gap-to-question 转换。

## Zotero 规则

- 优先通过 Zotero Connector 保存网页/数据库条目，或导入 RIS、BibTeX、CSL JSON 等标准格式。
- 为论文建立主题 collection，可用标签表示 `to-read`、`abstract-read`、`fulltext-read`、方法和文献簇。
- 导入后核对作者、年份、题名、刊名、卷期页、DOI/URL、摘要和附件；Zotero 条目存在不等于元数据正确。
- 检查重复项并采用合并而非简单删除，以保留 collection 和标签关系。
- Zotero item key、citation key、导入数量和 collection 名称必须来自实际工具结果，不能编造。
- 官方操作可核验 [Zotero 标准格式导入](https://www.zotero.org/support/kb/importing_standardized_formats)、[Collections and Tags](https://www.zotero.org/support/collections_and_tags) 与 [Duplicate Detection](https://www.zotero.org/support/duplicate_detection)。

## 来源状态的可用范围

- `candidate source`：只能说明待筛选。
- `metadata only`：只能确认书目信息和粗略相关性。
- `abstract only`：只能谨慎概括摘要明确写出的信息，并标注状态。
- `full text read`：可以使用全文支持的方法、发现、局限和准确转述。
- `in Zotero` / `imported to Zotero`：只表示库状态；仍需另标阅读状态。

详细发现、直接引语、页码和方法细节必须来自全文或用户提供的原文。不能用元数据或搜索摘要补齐。

## 交付物

- 检索策略与完整检索式
- 来源筛选日志和排除理由
- Zotero collection/导入/重复项报告（实际执行时）
- 结构化阅读笔记
- 文献综述矩阵和 gap statement
- 按文献簇组织的综述草稿
- 引文与证据风险报告

使用 `assets/literature-review-matrix-template.md`。如果写入用户项目，可按用户要求生成检索策略、筛选日志、阅读笔记、矩阵、综述草稿和参考文献文件；不要默认制造一组空文件。
