# 教育技术学论文功能性表达库

用于帮助章节承担明确功能，不用于模仿特定作者、替换证据或制造“C 刊腔”。表达是否可用取决于其前后的事实、理论与证据。

| 写作功能 | 可选表达 | 使用条件 | 常见误用 |
| --- | --- | --- | --- |
| 提出问题 | “在……活动中，……仍是尚未解决的具体问题。” | 已说明对象、情境与矛盾 | 只写“随着技术发展，问题日益突出” |
| 压缩缺口 | “现有研究已说明……，但对……仍缺少能够区分……的证据。” | 缺口可定位到解释、比较、过程或测量 | 用“研究较少”替代缺口 |
| 引出理论 | “为解释……，本文借助……界定……并推导……” | 理论将进入变量、活动、编码或解释 | 只报理论名称 |
| 说明设计 | “基于……材料，采用……以识别/比较/描绘……” | 方法功能与问题匹配 | 把软件或算法名写成方法贡献 |
| 归并发现 | “从……维度看，……共同构成识别/理解……的关键信息。” | 原始变量已按理论或观察含义归类 | 把特征重要性排序直接写成变量的净效应 |
| 报告结果 | “结果显示，在……范围内，……呈现……” | 有可定位表图或分析输出 | 只说“结果显著”或堆砌数字 |
| 概括过程结构 | “据此，……可概括为……相互联动的过程结构。” | 已区分数据事实、模型发现与理论解释；构念关系有理论或观察框架支撑 | 把解释性综合写成数据直接证明的因果链或发生顺序 |
| 解释收束 | “这一发现提示……可从……理解；其适用仍取决于……” | 区分结果与解释 | 把提示改写为已验证机制 |
| 限定边界 | “该结论适用于……；对于……仍需……” | 边界会改变结论强度或适用范围 | 在每段重复免责式声明 |
| 转向意义 | “该分析为……的维度优化、过程解释或后续验证提供经验依据。” | 意义可回溯到发现，且未承诺干预效果 | 直接提出刚性处方、保证性效果或政策口号 |

## 使用规则

1. 优先改写句子承担的功能，再选择表达；不因某一词出现就自动删除。
2. “影响、机制、形成、共同”等词可用于问题界定、理论讨论或解释性综合；只有作为本研究实证已确认的结论时才核对识别强度。
3. 连续使用同一种连接词、对称句式或“本文首先/其次/最后”时，先检查结构是否真的需要分项，而不是随机替换词语。
4. 面向教育技术与教育学期刊摘要时，优先使用“问题—发现簇—解释性综合—有限意义”的功能推进；“模型稳定依赖”“当前留出样本”等术语留给方法审计，除非它们确实改变某一摘要承诺。
