# 跨 Skill 工件路由

当研究任务需要图、表、DOCX、PDF、工作簿或非数据型配图时，先由本 Skill 完成研究判断，再按需调用当前环境中**已可用**的专项 Skill。不要因为“能画图”就增加图，也不要把专项 Skill 的输出直接当作研究结论。

## 路由原则

1. 先回答工件服务于哪项判断：解释理论关系、呈现研究设计、报告数据结果，还是满足投稿交付。
2. 先明确图表中的每个节点、箭头、数值、分组和标题各由什么材料支持；不能支持的关系改为研究问题、假设、待检验路径或删除。
3. 仅在当前环境确有对应 Skill 时调用；不可用时，诚实交付可交给人工或其他工具执行的图表说明书、数据表或版式规格，不声称已生成或核验成品。
4. 专项 Skill 负责工件制作和格式验证；本 Skill 负责论证、证据、图文一致性、图题图注和结论边界。

## 常用协作路由

| 需要解决的问题 | 优先调用的能力 | 调用前必须提供 | 返回本 Skill 后必须检查 |
| --- | --- | --- | --- |
| 研究框架、机制路径、DBR 迭代、研究设计流程、数据处理流程图 | `drawio` | 图的用途、节点/箭头含义、证据状态、层级、目标格式与保存位置 | 每一箭头是否被误作因果；图题、图注、正文首次引用和术语是否一致；图是否真正帮助回答研究问题 |
| 数据分布、趋势、组间比较、模型性能、SHAP/ALE 等结果图 | 当前可用的数据分析、表格或可视化 Skill | 已核验的数据、分析单位、统计口径、模型版本、图中数值和不确定性要求 | 不能把探索性图写成确认性发现；坐标、样本量、误差线、分组和正文数字一致；不把 SHAP/ALE 解释为因果效应 |
| 可编辑 Word 稿、匿名稿或排版后的终稿 | 当前可用的 DOCX/文档处理 Skill | 定稿文本、目标期刊官方格式、匿名要求、图表和参考文献位置 | 文本、图表、交叉引用、匿名信息、页眉页脚和最终渲染同步 |
| PDF 阅读、审稿件核对或终稿渲染检查 | 当前可用的 PDF 处理 Skill | PDF 文件、核对重点、目标页或工件清单 | 可读性、页码、图表清晰度、裁切、匿名和引用对应关系 |
| Excel/CSV 数据清理、结果表或附录数据表 | 当前可用的工作簿/表格 Skill | 数据字典、缺失规则、分析计划、变量显示名和脱敏要求 | 分析代码/模型输出与表格一致；不在表中制造不存在的显著性、样本或小数精度 |
| 封面图、仓库横幅或非结果型教学示意图 | `imagegen` 等图像生成能力 | 用途、非实证性质、视觉主题、版权和标注要求 | 不将生成图伪装成研究数据、课堂观察证据或研究结果图 |

## Drawio：论文图的协作协议

在用户要求研究框架图、流程图、机制图、模型图或数据流程图时，按以下顺序执行：

1. 先读取 `references/framework-defense-and-figure-audit.md`，输出图表说明书：图号/暂定图题、图的唯一判断、节点定义、箭头类型、证据状态、图注、正文放置位置。
2. 调用当前环境中可用的 `drawio` Skill，使用说明书制作可编辑图；优先保留 `.drawio` 源文件，并按用户需要导出图片。
3. 若环境还提供面向学术出版的 Drawio 扩展，论文、学位论文或期刊成图优先交给该扩展处理；若不存在，不要声称已执行其出版规范，只按目标期刊官方要求复核尺寸、字体、分辨率、图注和格式。
4. 生成后回到本 Skill，逐项执行图文一致性审计：节点术语、箭头动词、图内数值、图题图注、正文引用、研究设计和结论措辞必须一致。

不要把并列变量直接连成过程链；不要用实线箭头暗示未经检验的因果关系；不要为了“图看起来完整”补造中介、反馈或闭环。

## 最小工件说明书

调用专项 Skill 前，至少给出：

```text
工件类型：研究框架图 / 结果图 / DOCX / PDF / 数据表
服务判断：该工件要帮助读者理解什么，而非仅展示什么
输入依据：已给材料、数据版本、理论或研究设计
元素规则：节点、箭头、分组、数值、图例各自的含义与证据状态
禁止表达：不可写成的因果、机制、效果或推广结论
交付要求：可编辑源文件、目标格式、保存位置、图题/图注和正文引用位置
验收条件：与正文、表格、附录和结论一致；可追溯；无虚构信息
```

若用户只要“画一张图”，先追问或合理限定图的服务判断；不能从缺失的理论、数据或结果自动补造节点和关系。
