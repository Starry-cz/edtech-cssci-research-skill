# 教育技术学论文论证与重构修订

用于搭建论文主线、重构全文、处理“有材料但写不成论文”、区分结果与讨论，以及将导师或审稿意见转为结构性修改。它沉淀的是可迁移的研究推理，不模仿某位作者的句式、题目或结论。

## 一、先建立主论证链

任何正文、模型或实证结果都应能回到下列链条：

```text
真实教育问题 → 可回答的问题 → 与问题相称的设计
→ 可核验的证据 → 有限定的回答 → 贡献 → 实践含义与边界
```

写作前先用一句话分别写出：

| 节点 | 必须回答的问题 |
|---|---|
| 问题 | 谁在何种教育活动中遇到什么可观察的困难，现有解释为何不足？ |
| 回答 | 本文最终要回答哪个关系、机制、差异、过程或设计原则？ |
| 证据 | 用什么材料、分析单位和方法支撑回答？ |
| 贡献 | 相比已有研究，本文具体修正、补充、区分或验证了什么？ |
| 边界 | 结论不适用于哪些对象、情境、时间范围或因果判断？ |

任一节点不能从论文材料中追溯时，不以更流畅的表述掩盖缺口，而标记 `[待补材料]`、`Evidence Gap` 或 `Conclusion Overreach`。

## 二、论文不是章节清单，而是连续回答

### 1. 三类常见论文路径

| 类型 | 主线 | 必须区分 |
|---|---|---|
| 实证研究 | 问题/假设 → 设计与数据 → 结果 → 解释 → 有限推广 | 关联、预测、因果；结果、解释、建议 |
| 模型与实践研究 | 真实难题 → 理论与构念 → 模型/系统 → 实施与证据 → 设计原则 | 提出、实现、可用、初步有效、机制成立、可推广 |
| 政策/比较研究 | 比较问题 → 共同维度与语料 → 情境化证据 → 共性/差异 → 条件化转译 | 政策目标、实施过程、实际效果 |

用户材料可以包含多条线，但每篇论文只设一条主回答。无法服务主回答的背景、案例、统计结果和图表，删除、压缩或移入补充材料。

### 2. 章节协议

写每一节前先明确“本节最小结论”。若不能写出最小结论，通常说明该节只是资料堆放。

| 部分 | 承担的判断 | 至少需要的材料 | 常见越界 | 优先修法 |
|---|---|---|---|---|
| 题目/摘要 | 问题、对象、方法、核心发现和边界是否一致 | 全文最终问题、方法与主要结果 | 把研究愿景写成已证实发现 | 先核对摘要每句能否在正文找到依据 |
| 引言 | 为什么这个具体教育矛盾值得回答 | 现象、对象、情境、已有研究不足 | 政策或技术热点替代研究问题 | 写出“主体—活动—失效环节—后果” |
| 综述 | 已有研究怎样推进、争论或留下可处理缺口 | 代表研究的对象、方法、发现和边界 | 作者排队或“研究较少” | 按争论、机制、方法或情境重组 |
| 理论/机制 | 理论为何能解释本问题，并如何进入分析 | 概念定义、关系命题、变量/编码/设计依据 | 理论只在综述中出现 | 建立“理论—部件—证据”映射 |
| 方法 | 为什么以这种材料和程序回答问题 | 样本/语料、分析单位、步骤、质量控制 | 软件或统计教材复述 | 逐项说明选择理由与不能支持的判断 |
| 结果 | 证据对研究问题说明了什么 | 结果表图、估计、不确定性、过程材料 | 逐表复述或提前解释原因 | 按“问题—证据—差异—最小判断”写 |
| 讨论/结论 | 结果改变、补充或限制了什么理解 | 对照理论、文献、设计条件与局限 | 由相关或案例跳到普遍机制 | 区分解释、贡献、建议和边界 |

## 三、用证据管理贡献强度

贡献不是“意义重大”的同义词。按照证据强度选择准确动词：

- **提出/界定**：形成概念、框架、命题或分析维度；不得写成“证明有效”。
- **描绘/识别**：基于描述、分类、调查或案例呈现模式；不得自动推成因果。
- **关联/预测**：报告统计关联或样本外预测；不得改写成干预效果或作用机制。
- **检验/支持**：在与问题相称的设计下支持特定命题；同时保留替代解释和误差。
- **形成设计原则**：由跨轮迭代、实施与过程证据归纳；必须交代适用条件。

出现下列情况标记 `Contribution Inflation`：系统上线即称效果、一个案例即称通用模式、政策文本即称实施成效、显著性即称教育价值、预测重要性即称因果作用。

## 四、全局到局部的重构顺序

先处理会改变其他章节的决定，再处理句段。默认顺序为：

1. 锁定研究问题、对象、分析单位和最小贡献。
2. 检查综述缺口、理论命题、模型部件与研究设计是否共同服务该问题。
3. 核对数据/材料、分析程序、结果图表与能够成立的结论强度。
4. 重排章节和段落功能；每段只承担一个可辨识的推进动作。
5. 最后修订标题、摘要、过渡、术语、图表文字和格式。

不要先“润色”引言或回应函，再发现问题、变量、样本或结论已改变。此类做法会制造 `Abstract Drift`、`Structure Drift` 和版本不一致。

## 五、修订连锁检查

任何实质修改都要追问其传播范围：

| 改动起点 | 必查连锁项 |
|---|---|
| 研究问题或范围 | 题目、摘要、综述缺口、理论、样本、结论与目标期刊匹配 |
| 理论、构念或模型 | 变量/编码、图示、方法、结果解释、贡献与实践建议 |
| 数据、样本、清理或模型 | 样本流向、表图、估计/不确定性、摘要、结论、附录和回应函 |
| 结果解释或结论强度 | 讨论、贡献、建议、因果措辞、局限与标题/摘要 |
| 图表或案例 | 正文读图逻辑、方法来源、证据状态、图注、匿名稿和附件 |

输出修订方案时使用“问题—首要修复层—修改动作—连锁项—验证”格式；不要只说“加强论证”。

## 六、建议与实践含义的生成规则

建议必须可回溯到发现，采用：

```text
基于[具体发现]，由[责任主体]在[场景]采取[行动]，
依赖[资源、规则或能力条件]，并用[过程或结果指标]跟踪。
```

对于模型、案例或预测研究，说明建议属于“待验证的设计方向”“基于关联的风险提示”还是“已有实施证据支持的行动”；不把建议写成政策口号。

## 七、输出与外部标准

- `argumentation_blueprint` 默认输出：主论证链、章节功能表、主张—证据—边界矩阵、阻断问题和下一步材料。
- `revision_cascade` 默认输出：修改优先级、首要修复层、连锁更新、验证动作与未决事项。
- 实证报告的透明性可参照 [AERA 经验社会科学研究报告标准](https://www.aera.net/Research-Policy-Advocacy/AERA-Shaping-Research-Policy/Standards-for-Research-Conduct/Standards-for-Empirical-Social-Science-Research) 与 [APA JARS](https://www.apa.org/journals/authors/all-instructions.html)。它们用于检查报告完整性，不替代目标中文期刊的当前官方要求。
