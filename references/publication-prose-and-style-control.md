# 投稿正文与方法审计的分离

用于把研究底稿转成可投稿的中文教育技术学正文。它不把“严谨”理解为把所有技术限制写进摘要、讨论和结论，也不模仿特定作者。

## 一、正文的优先级

正文首先让读者看见教育问题、理论视角、研究行动、发现和贡献。方法透明性由方法、表图、附录和必要的补充材料承担；它们不是每段正文都要重复的防御性语句。

| 位置 | 应优先呈现 | 不宜默认呈现 |
| --- | --- | --- |
| 摘要 | 问题、材料/行动、发现簇、中心判断 | 切分、调参、解释器、稳定性和未来验证清单 |
| 结果 | 经验发现、关键比较和图表证据 | 对每项数值重复技术限定 |
| 讨论 | 理论对话、过程解释、教育含义 | “模型只能说明什么”的反复说明 |
| 结论 | 对题目的回答、贡献和实践启示 | 完整方法审计或未来研究愿望清单 |
| 局限 | 一至两项最影响适用范围的研究条件 | 把所有可能的技术问题一次性罗列 |

## 二、何时才进入方法审计

仅在用户要求数据/代码复核、研究设计诊断、投稿前方法预审，或已知事实与正文主张直接冲突时，另行输出审计说明。审计需具体定位材料、说明它影响哪一项承诺并给出修复动作。

例如，同一测试集参与候选模型比较，主要影响“独立最终最优性能”的说法；它不自动要求将摘要改成“当前留出样本”，更不要求删除关于课堂、学习或评价的教育学判断。缺少因果设计时，优先在实证结论中避免把单一行为写成已被证明的净效应；不必删除理论讨论中关于形成逻辑、过程结构和实践改进的正常表达。

## 三、保留自然作者声音

- 允许使用“形成、塑造、机制、共同、何以达成、影响”等领域语言；结合标题、理论段、实证结论和文类判断其强度，不制作机械禁词表。
- 允许“本文”“本研究”“第一、第二”等有真实组织功能的表达；修复的是无主体、过密或为了整齐而重复的句式。
- 让转折来自事实、理论分歧或材料差异，不用口号、对偶或连续元话语代替论证。
- 系统、图表和模型只承担其实际支持的判断；不要反过来让一串免责声明取代讨论和结论。

## 四、最终提交契约

终稿不允许以“投稿时补充”“应如实说明”“待完善”“按要求填写”等文字替代编码、授权、伦理、来源、作者、基金、图表或期刊必填事实。事实未齐时，交付可用正文与稿外清单，不称为最终稿。

## 五、表层检查

清洁稿不出现“目的：/方法：/结果：/结论：”构思标签（期刊要求结构式摘要除外）、RQ编号、模式名、风险等级、待办或占位符。完成稿件时可运行 `scripts/audit_manuscript_surface.py <稿件> --final`；它只扫描占位候选，不替代学术判断。
