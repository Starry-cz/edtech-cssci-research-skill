# 投稿正文表层与学术表达控制

用于把论证底稿、诊断标签和方法检查转化为可投稿的中文教育技术学正文。适用于摘要改写、全文生成、段落润色和终稿自检。它控制的是“哪些思考只留在内部、哪些内容可以进入论文”，不以模仿某位作者的句式为目标。摘要的产物类型与信息密度另见 `abstract-state-and-evidence-control.md`。

## 导航

- 双层写作与发布门
- 题目和摘要的表层控制
- 引言、方法、结果与讨论的证据边界
- 结论、建议与自然作者声音
- 表层放行扫描及规范依据

## 一、坚持双层写作

### 1. 内部构思层

内部可以使用问题、目的、对象、方法、结果、贡献、边界、证据状态、风险标签和修订优先级。它们用于防止遗漏、核对证据和安排章节，不是默认的论文措辞。

### 2. 投稿正文层

正文应把上述维度转化为连续的学术叙述。除非目标期刊的官方投稿说明明确要求结构式摘要、编号研究问题或固定栏目，不要把下列内容直接输出到论文：

- “目的：”“方法：”“结果：”“结论：”等构思标签；
- `RQ1`、`argumentation_blueprint`、`Evidence Gap`、`Pass/Fail` 等工作流标记；
- `[待补数据]`、`[needs user verification]`、`投稿前补充` 等占位符；
- “理论层面—方法层面—实践层面”三格贡献模板；
- 为了显得完整而生成的“一是、二是、三是”对称清单；
- 对方法步骤、软件、模型和图表的目录式复述。

缺失信息放入修改说明、审计报告或待办清单，不得混入准备提交的清洁稿。

## 二、正文生成前先做风险判断，按需提示

出现下列问题时，先在内部判断它会影响哪一项主张。用户要求投稿判断、方法诊断或定稿核验时，先输出诊断与修复方案；用户只要求改写摘要、段落或结构时，应在不掩盖风险的前提下完成相称改写，而不是用方法审计替代写作。

1. 题目使用“影响、作用、机制、塑造、何以达成、提升”等强因果词，但设计只能支持描述、关联或预测。
2. 目标变量与特征共享题项、评分规则、观察维度或评分者，尚未排除准则污染或构念循环。
3. 插补、标准化、降维、特征选择或重采样早于数据切分，或同一教师、班级、学校的记录跨训练与测试。
4. 用同一测试集比较大量模型、选择最优模型并报告其为最终样本外表现，没有独立确认或嵌套验证。
5. 结果尚未稳定，却已经生成确定的阈值、机制、实践处方或贡献结论。
6. 关键样本来源、测量、伦理、数据权限或结果数字无法核验。

风险判断不要求研究完美；它要求论文表层准确反映当前证据状态。探索性分析可以自然地写成当前发现；只把会改变读者理解的限制集中放在摘要、讨论或稿外修改说明的一处，不反复插入“不能构成因果”之类的声明。

### 发布状态对应的表层权限

| 状态 | 可写的正文 | 不可写的正文 |
| --- | --- | --- |
| `阻断` | 诊断报告、重跑方案，以及不超出当前证据的自然摘要、结果或讨论文本。 | 把未修复风险隐去后写成投稿定稿；最终样本外表现、已确认因果效果或刚性实践处方。 |
| `探索性可写` | 限定性的描述、关联、预测、模型诊断段和与理论相连的解释性讨论。 | 已确认的因果效果、普遍推广，或将未经验证的解释写成实证机制。 |
| `投稿就绪` | 连续摘要、清洁正文和提交前表层审计。 | 内部工作标签、未经官方核验的格式要求、超出证据的动词。 |

摘要生成前使用 `assets/abstract-argument-card-template.md` 和 `abstract-state-and-evidence-control.md`。论证卡只用于验证承诺，不作为论文附表或投稿附件。

## 三、题目：先匹配识别强度

| 设计与证据 | 安全的题目关系词 | 需要额外证据才使用 |
| --- | --- | --- |
| 横断面描述、观察编码 | 特征、样态、分布、差异、关联 | 形成、作用、机制 |
| 预测建模、SHAP/ALE | 预测、预测线索、模型依赖、非线性预测模式 | 塑造、促进、驱动、何以达成 |
| 纵向或面板设计 | 变化、发展轨迹、时序关联 | 长期效应、发展机制 |
| 实验或可信准实验 | 效果、影响、作用 | 普遍机制、必然路径 |
| 案例、模型或系统构建 | 构建、应用、过程、初步检验 | 有效模式、可复制推广 |

题目中的每个强动词都要在方法和结果中找到识别依据。标题修辞不能替代研究设计。

## 四、摘要：维度要齐，标签不必出现

### 1. 内部核对维度

内部检查问题、对象与情境、材料或设计、核心发现、解释或实践含义、边界是否齐全。该结构用于验收内容，不等于逐项加标签。

### 2. 默认表层顺序

中文教育技术学论文的非结构式摘要可自然完成以下移动：

1. 用一至两句指出具体教育问题或解释缺口。
2. 用一句压缩对象、样本/材料、设计和方法功能，不堆模型名称。
3. 按主次而非分析步骤呈现一项中心发现及必要的次级发现；发现数量由主回答决定，不按“四项分析”凑数。
4. 用一句给出由发现直接支持的认识或条件化实践含义。
5. 只有对防止误读必要时，才在摘要中补一处边界。

这些移动应连成一段。允许使用“研究发现”“结果表明”“据此”等自然衔接，但不要把摘要写成实验报告表单。

### 3. 摘要压缩原则

- 优先保留研究对象、关键样本、最能回答问题的结果及其含义。
- SHAP、ALE、Bootstrap、消融、软件和超参数只保留对读者理解设计不可缺少的部分；其余放入方法。
- 不在摘要中同时重复模型名、指标名、排序、方向、阈值、稳定性和四层贡献。
- 数值、模型名和验证信息不因“更像教育学”而一律删除。只保留能够改变读者对中心回答可信度或适用范围判断的锚点；性能是主回答时可保留一个相称指标，性能并非主回答时应转入正文表图。
- 先删去算法目录、重复指标和按代码顺序展开的结果；若删去后读者已无法判断研究为何可信，补回说明研究类型或验证角色的最小锚点，而不是重新列出全部技术步骤。
- “贡献率”必须写清是对模型预测归因的份额还是结果变量的解释比例；二者不能互换。
- 预测研究的实证结论优先使用“模型更依赖”“预测重要性较高”“呈现预测关联”等准确表达；“形成、影响、机制、共同”等词并非禁词，但须区分它们是在提出问题、回顾理论，还是宣称本研究已验证因果关系。

### 4. 结构式摘要例外

只有目标期刊官方说明明确要求时，才保留“目的/方法/结果/结论”等小标题。APA 的通用报告标准强调内容完整，但具体期刊可能另行要求结构式摘要；因此格式必须按目标期刊核验，而不能由写作模板自行决定。

## 五、引言与研究问题：把设计语言藏在论证后面

- 引言先形成“现实矛盾—已有解释—可处理缺口—本文入口”，不从政策口号和方法优势直接跳到选题。
- 研究问题要能被材料回答，但不强制写成 `RQ1—RQ4`。中文期刊范文常把问题整合进研究目的、框架或段末；只有多研究设计、假设检验、目标期刊惯例或用户明确要求时才编号。
- 不用“本文旨在填补空白”代替缺口说明。写清已有研究在哪个对象、过程、比较、测量或识别层面无法回答当前问题。
- 不为迁就已有数据事后拼接理论。若分类只用于整理观测变量，称“特征分类/分析框架”，不要自动称为理论框架。

## 六、方法：报告研究决策，不复述教材

每个方法段回答四件事：为什么需要该步骤、该步骤使用什么材料、它生成何种证据、它不能支持什么判断。公式、算法沿革和术语定义只在帮助复核时保留。

对于机器学习论文，先写数据生成与切分，再写预处理、验证和选模，最后写解释。若方法流程未通过发布门，应明确提出重跑要求，不用局限段替代修复。

## 七、结果：从证据推出最小判断

- 结果小标题可以是发现式标题，但强度不得超过证据。
- 先回答对应问题，再给关键数值、比较对象和不确定性；表中已有的全部数字不必在正文重报。
- SHAP 表示模型对信息的依赖，不是变量对教育结果的净贡献；ALE 是模型预测函数的局部形态，不是现实因果效应。
- 多模型性能接近且区间重叠时，写“表现相近”或“并列候选”，不按小数点偶然领先宣布唯一最优。
- 分组样本小、尾部密度低或解释不稳定时，停止排名和阈值命名。

## 八、讨论：不把并列特征当作已验证的过程机制

讨论按“核心发现—与理论/文献的关系—替代解释—教育含义—边界”推进。只有证据中存在时间顺序、行为序列、交互结构、过程追踪、实验操纵或质性过程材料时，才能把多个变量组织成“链条、闭环、路径、阶段或机制”。

下列推断默认不成立：

- 多个特征同时重要，所以它们构成先后发生的过程闭环；
- SHAP 排名靠前，所以该因素是教育结果的决定性原因；
- ALE 曲线转折，所以发现了可直接执行的教学阈值；
- 删除特征后性能下降，所以移除该教育行为会导致成效下降；
- 同源观察指标能高精度预测总分，所以模型识别了真实课堂质量机制。

若理论上希望提出机制，可将其写为“由结果启发的解释性综合”或“待检验解释”，并说明还需什么时间、过程或干预证据。不要把这种解释性综合误写成模型已经直接观察到的先后过程。

## 九、结论与建议：不机械复述全文

- 结论优先给出一个总判断，再选择最能支撑它的两至三项证据，不按研究问题逐条复制结果。
- 不强制列“理论、方法、实践”三类贡献。只保留确有证据的贡献，并写明相对于哪类已有研究推进了什么。
- 建议必须从具体发现回溯，说明责任主体、行动对象、适用条件和验证指标。预测线索只能支持诊断优先级或待检验干预方向，不能直接生成刚性评价门槛。
- 局限要改变读者对结论强度或适用范围的理解，不把“未来扩大样本”当作万能结尾。

## 十、保留自然作者声音

- 不要机械清除“本文”“本研究”。在说明作者选择、研究目的、材料边界和贡献时适量使用，通常比“分析提出”“结果自行表明”等无主体表达更自然。
- “第一、第二”“一方面、另一方面”并非禁词。只有在没有真实分类依据、段落机械对称或连续高频出现时才重写。
- 专业术语、真实数字、引用和必要的逻辑衔接应保留；去模板化不等于口语化、删术语或随机改句式。
- 少造新标签、口号和对偶结构。新概念必须有定义、关系、操作化与可证伪后果。

## 十一、表层放行扫描

交付清洁稿前逐项搜索并判断：

| 搜索信号 | 默认处理 |
| --- | --- |
| `目的：`、`方法：`、`结果：`、`结论：` | 无官方结构式摘要要求时删除标签并重组为连贯摘要 |
| `RQ1`、`RQ2` | 核验期刊/论文类型是否需要；否则融入自然问题陈述 |
| `待补`、`待核验`、`Evidence Gap`、`Pass/Fail` | 移出清洁稿，放入修改说明 |
| 连续出现“理论层面/方法层面/实践层面” | 检查是否为真实贡献，删除凑数项并改为具体推进 |
| `闭环`、`链条`、`路径`、`机制` | 判断它是理论解释、研究问题还是实证结论；只有后者才回查顺序、过程、交互或因果证据 |
| `贡献率`、`影响因素`、`决定性` | 回查指标含义和识别强度，补全“对模型预测”或降级 |
| `最优模型`、`样本外` | 回查是否隔离了选模集与最终测试集，并报告不确定性 |
| 高频“本文/本研究”或完全没有研究主体 | 调整到必要位置，不作一刀切删除 |

## 十二、依据与使用边界

- [AERA 经验社会科学研究报告标准](https://www.aera.net/Publications/Standards-for-Research-Conduct)用于检查教育研究报告的完整性与透明性，不规定中文期刊的固定句式。
- [APA 通用稿件准备说明](https://www.apa.org/journals/authors/all-instructions.html)将 JARS 作为报告标准，并要求继续核对具体期刊说明。
- [APA 同行评审资源](https://www.apa.org/pubs/journals/resources/peer-review)强调摘要应突出发现及其含义，可用于检查摘要是否结论导向。
- [TRIPOD+AI](https://www.equator-network.org/reporting-guidelines/tripod-statement/)可为预测模型透明报告提供跨领域启发，但其原始适用范围是临床预测模型，不能写成教育技术学期刊的强制规范。

领域范文用于观察问题推进、章节功能和表层表达，不作为方法正确性的唯一依据，也不复制作者句式、题目、论断或长段文本。
