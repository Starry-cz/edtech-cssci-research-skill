# 选题—研究问题—证据画布

用于教育技术学 CSSCI/中文核心研究的选题诊断与研究问题细化。先完成画布，再写大纲或摘要。

| 环节 | 需要填写 | 证据状态/来源 | 不能作出的判断 |
| --- | --- | --- | --- |
| 外部现象 | 当前技术、政策、实践或研究趋势是什么？ |  | 热点本身不等于问题 |
| 真实教育问题 | 谁在什么活动中遇到何种可观察困难、差异或风险？ |  | 不用“应用不深”替代问题 |
| 对象与情境 | 教育层级、课程、任务、主体、地区/制度与时间范围 |  | 不将单一场景推广到所有场景 |
| 技术角色 | 技术改变什么任务、反馈、协作、评价或决策？有何替代方案？ |  | 功能不等于机制或效果 |
| 理论与机制 | 哪些构念、过程、条件和替代解释需要检验？ |  | 新术语不等于理论贡献 |
| 证据可得 | 已有/可收集的需求、过程、实施、结果、比较或文本材料 |  | 不为方法猜测数据 |
| 主研究问题 | 用一句可回答的问题界定对象、关系/过程、情境与边界 |  | 不写复合口号式问题 |
| 子问题 | 至多三个，分别承接机制、条件、设计或结果 |  | 子问题不应各自需要一项独立研究 |
| 方法与报告 | 每题对应的单位、时间、设计、主要分析与报告规范 |  | 不以显著性/预测替代设计判断 |
| 最小贡献与边界 | 本研究最多能新增什么认识，对何处不外推？ |  | 不写“具有重要意义”作为贡献 |

## 快速判定

- 若“真实教育问题”“证据可得”或“主研究问题”为空：停止命题，先补材料。
- 若主问题出现“如何促进/赋能/提升”但没有比较、机制或过程：改为解释、设计或描述问题。
- 若结论预计依赖单次问卷、案例或平台指标：在问题中降低因果或推广主张。
- 若涉及生成式 AI、学习分析或学生数据：补填人工判断、数据用途、偏差/受损群体、错误纠正与申诉安排。
