# 教育技术学研究定位与外部证据矩阵

用于将报告、期刊范围和公开学术共同体资料转为研究问题、设计条件与边界。外部资料只支持情境、概念和条件判断；本研究的发现与结论必须另列本研究证据。

| 层级 | 需要回答的问题 | 已有材料与状态 | 可支持的判断 | 仍需补充或核验 |
| --- | --- | --- | --- | --- |
| 教育问题 | 谁在何种活动中遇到何种可观察困难？ |  |  |  |
| 情境与对象 | 教育层级、课程、群体、资源与制度条件是什么？ |  |  |  |
| 技术角色 | 技术具体改变什么任务、反馈、协作、评价或决策？有哪些非技术替代？ |  |  |  |
| 理论机制 | 为什么该变化可能影响学习、教学或治理？有哪些替代解释？ |  |  |  |
| 外部定位 | 报告、期刊范围或公开资料提示了什么问题/条件？ |  |  |  |
| 研究设计 | 分析单位、时间、比较/迭代、数据与伦理安排如何对应问题？ |  |  |  |
| 过程证据 | 技术、主体、活动和情境实际如何互动？ |  |  |  |
| 结果证据 | 哪个结果对应哪个研究问题？不确定性和替代解释是什么？ |  |  |  |
| 行动与治理 | 谁根据结果采取什么行动？权限、偏差、申诉、维护和资源条件是什么？ |  |  |  |
| 贡献与边界 | 最小可支持贡献是什么？对哪些对象/情境不作外推？ |  |  |  |

## 使用规则

1. 在“已有材料与状态”中标记 `已核验`、`待核验` 或 `待补材料`，并写来源类别：本研究数据、原始文献、官方报告、期刊公开范围或用户材料。
2. 不将报告中的宏观判断、期刊范围或系统功能直接填入“结果证据”。
3. 若技术功能、理论机制或行动主体为空，先回到研究问题与设计；不要用“赋能”“重塑”等动词填补。
4. 当研究涉及未成年人、学习分析或生成式 AI 时，至少填写数据用途、权限、人工复核、错误纠正和潜在受损群体。
