## 2. 条件性的验证与风险检查

- 当前状态：`阻断 / 探索性可写 / 投稿就绪`
- 本轮产物类型：`诊断与修复计划 / 当前版本摘要 / 投稿摘要`
- 目标期刊官方摘要要求核验状态：`已核验 / 待核验`
- 会改变当前摘要**具体句子**的风险或待修复项：
- 是否需要在稿外提示风险：`仅投稿判断时 / 风险会改变当前主张时 / 不需要`

仅当摘要拟使用“最终最优、独立泛化、严格阈值、净效应、已验证机制、刚性处方”等强承诺时，才把相应验证状态作为摘要的必要锚点；普通教育问题、构念归并和解释性过程结构不以内部审计语言替代。

## 3. 预测/可解释机器学习专用检查

- 训练、验证与最终测试的角色是否分开：
- 是否用测试集选模、早停、阈值或解释模型：
- 目标—特征重叠、群组/时间泄漏和折外预处理是否已排除：
- 摘要中的发现属于：`性能 / 模型归因 / 教育解释 / 行动建议`（分别填证据）：
- 是否区分了这些词是在研究问题、理论解释还是实证结论中使用；仅降级超出设计的实证承诺：
- 是否把 SHAP、ALE、节点删除分别限定为模型归因、预测曲线和模型依赖：

## 4. 连续摘要放行检查

- [ ] 没有把内部标签、RQ 编号、风险标签或占位符放进摘要。
- [ ] 方法只保留理解研究设计所需的最小信息，不罗列全部模型、解释器和诊断。
- [ ] 每项结果均可回指正文表图或经核验的分析输出。
- [ ] 原始变量已按理论或观察含义归并；摘要优先写过程维度和中心判断，不复述特征排行榜。
- [ ] 教育学解释与模型发现相邻，但明确属于解释性综合，不伪装为模型直接观测到的因果机制。
- [ ] 结尾贡献只复述当前证据能够支持的认识或条件化含义。
- [ ] 未把“更教育化”的概括写成因果、机制、阈值或普遍行动处方。
- [ ] 目标期刊的摘要格式、字数与关键词要求已由当期官方页面核验。
