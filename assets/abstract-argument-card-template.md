# 摘要论证卡（内部工作底稿）

用于写前压缩，不进入论文或投稿附件。填写后只保留能服务读者判断的内容，生成连续摘要。

## 1. 中心判断

- 要回答的教育问题：
- 研究对象与情境：
- 数据/材料与研究行动（用一句话）：
- 中心发现（只写一项）：
- 次级发现簇（最多三项，按构念或过程归并）：
- 理论或实践上的推进：

| 原始变量/编码 | 教育构念 | 过程维度或比较类型 | 摘要中是否需要出现 |

不要把变量重要性清单直接搬入摘要。先说明它们共同回答了什么教育问题，再保留最有代表性的发现。

## 3. 信息取舍

- 必须保留的样本、情境或方法信息：
- 必须保留的数字（若有）及其作用：
- 转入方法、结果表图或附录的技术细节：
- 需要在全文局限处处理、但不进入摘要的研究条件：

## 4. 连续摘要放行检查

- [ ] 第一段或首句进入的是具体教育问题，不是算法介绍。
- [ ] 方法只说明研究行动，不是模型、软件和检验目录。
- [ ] 结果按教育发现组织，而不是按代码执行顺序组织。
- [ ] 末句给出明确的认识推进或实践意义，不以通用免责语收束。
- [ ] 摘要不含内部标签、风险等级、待办或占位符。
- [ ] 若需要方法审计，审计说明另附，不混入摘要。
