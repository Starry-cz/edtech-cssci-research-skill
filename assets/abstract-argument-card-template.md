## 1.1 信息取舍

- 摘要的中心回答（只写一项）：
- 需要保留的次级发现（通常一至数项，不按“四发现”凑数）：
- 必须转入正文/附录的方法或结果细节：
- 摘要是否需要性能指标；若需要，说明其验证角色和不确定性：
- 每一项拟保留的技术术语/数字是否改变读者对研究类型、验证角色、中心发现可信度或适用范围的判断：
- 先删去算法目录与重复数字后的摘要骨架，是否仍能回答“为什么研究、发现什么、意味着什么”：

## 2. 预测/可解释机器学习专用检查

- 训练、验证与最终测试的角色是否分开：
- 是否用测试集选模、早停、阈值或解释模型：
- 目标—特征重叠、群组/时间泄漏和折外预处理是否已排除：
- 摘要中的发现属于：`性能 / 模型归因 / 教育解释 / 行动建议`（分别填证据）：
- 是否删除了“最优、机制、何以达成、塑造、阈值、处方”等超出设计的表达：
- 是否把 SHAP、ALE、节点删除分别限定为模型归因、预测曲线和模型依赖：

## 3. 连续摘要放行检查

- [ ] 没有把内部标签、RQ 编号、风险标签或占位符放进摘要。
- [ ] 方法只保留理解研究设计所需的最小信息，不罗列全部模型、解释器和诊断。
- [ ] 每项结果均可回指正文表图或经核验的分析输出。
- [ ] 结尾贡献只复述当前证据能够支持的认识或条件化含义。
- [ ] 未把“更教育化”的概括写成因果、机制、阈值或普遍行动处方。
- [ ] 目标期刊的摘要格式、字数与关键词要求已由当期官方页面核验。
