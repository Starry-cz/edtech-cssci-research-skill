# 摘要写作提示模板

## 需要输入

研究问题、对象/情境、材料或设计、已核验的中心发现、可直接支持的认识、目标期刊要求（如已核验）。

## 本节承担的判断

让读者迅速理解“为何研究、研究了什么、发现了什么、意味着什么”。教育技术与教育学的非结构式摘要优先呈现研究判断，而不是复述算法、代码或全文目录。

### 默认组织（按研究问题调整，不是固定句式）

1. 具体教育问题与缺口；
2. 对象、材料/设计与分析框架的最小交代；
3. 归并后的核心发现簇，而非变量或模型操作的流水账；
4. 基于理论和观察维度的教育学解释：可概括为怎样的过程结构；
5. 对评价、教学、设计或后续研究的有限意义。

预测研究的第四步允许形成解释性综合，但不能把它写成模型已识别的因果机制或教学处方。

## 内部核对

- 方法、数字和术语只保留会改变研究类型、验证角色或中心发现判断的信息。
- 区分实证结论、理论解释和问题措辞；不把预测归因改写为已验证因果。
- 先完成“原始变量 → 构念 → 过程维度 → 教育学解释”的压缩；发现列表不能代替中心判断。
- 验证风险只收束相应强承诺：不把“当前留出样本”“模型稳定依赖”“重跑步骤”等审计语言默认搬入摘要。
- 优先通过改写强承诺本身处理边界，而非在摘要末尾添加“非因果/当前版本”等免责声明；只有不加限定就会实质误导读者时，才在摘要中保留一处最小限定。
- 仅在风险会改变摘要核心主张时，于稿外集中提示一次；不默认添加免责声明。

## 输出契约

输出一段连续自然的中文摘要；目标期刊已核验结构式摘要要求时才添加标签。另列不进入正文的修改说明仅在用户要求诊断或投稿判断时提供。
